## 2. Regression locks (per-defect: mechanism, snippet, lane, cost)

All locks below are **ExUnit meta-tests in the default suite** unless noted, so they ride the **merge-blocking `quality` lane** at **~0 added wall-clock** (string reads + regex). This matches v1.20's async-safety AST guard and the install_smoke parity family exactly.

### Lock A — phx.new availability (defect 1)

Two complementary parts; **ship both** (assert-in-script AND cold-path exercise). The script guard is the durable correctness fix; the meta-test locks that the guard text stays present.

**A1 — meta-test asserting the guard literal (merge-blocking, ~free).** Drop into `test/install_smoke/package_metadata_test.exs` (it already reads `install_smoke_script`) or a new `install_smoke_preflight_test.exs`:

```elixir
test "install_smoke.sh self-installs the phx.new archive before the generated-app smoke (cold-runner guard)",
     %{install_smoke_script: script} do
  # The generated_app smoke runs `mix phx.new`; on a COLD runner the archive is
  # absent and the smoke would fail. Lock the idempotent self-install guard so it
  # can never be silently deleted (regression of the 2026-06-26 warm-cache-only pass).
  assert script =~ "mix phx.new --version",
         "install_smoke.sh must probe for the phx.new archive before using it"
  assert script =~ "mix archive.install hex phx_new --force",
         "install_smoke.sh must self-install the phx.new archive when absent"
  # Order matters: the probe/install must precede the `mix test ...generated_app_smoke_test`.
  install_idx = :binary.match(script, "mix archive.install hex phx_new") |> elem(0)
  smoke_idx = :binary.match(script, "generated_app_smoke_test.exs") |> elem(0)
  assert install_idx < smoke_idx,
         "phx.new archive must be installed BEFORE the generated-app smoke runs"
end
```

**A2 — make the cold path actually exercised in CI (preferred hardening).** The current `package-consumer` lane reuses a warm runner; a deleted guard would still pass there. Add a **runtime preflight inside the script** that fails LOUDLY (not silently self-heals) is the wrong call — self-heal is correct for adopters. Instead, exercise the cold path: in the `package-consumer` job, run the archive-purge before the smoke so the self-install is genuinely taken:

```yaml
# ci.yml package-consumer job, BEFORE "Run built-artifact image-only package-consumer proof"
- name: Purge phx.new archive to exercise the cold-runner self-install path
  run: mix archive.uninstall phx_new --force || true
```

This is a 1-line, ~0-cost CI step that converts the warm-only smoke into a real cold-path proof every PR. **Lane: `package-consumer` (merge-blocking, on all triggers).** Cost: negligible (the self-install is one small hex archive, already paid on cold runners).

> **Footgun avoided:** a regression test that itself depends on the warm cache reproduces the very bug it guards. A2 deletes the cache precondition so the guard is honest.

### Lock B — `:focus-visible` keyboard-modality assumption (defect 2)

**Recommendation: DEDUPE first (root fix), then a single dual-presence meta-test as belt-and-suspenders.** See §3 for the dedupe rationale. The lock has two layers:

**B1 — meta-test asserting the Tab-first pattern is present in every copy that drives `:focus-visible` (merge-blocking, ~free).** Until dedupe lands (or permanently, if dedupe is rejected), a meta-test that the keyboard-modality press precedes the programmatic focus in BOTH harness files:

```elixir
defmodule Rindle.FocusVisibleModalityGuardTest do
  use ExUnit.Case, async: true
  @harnesses [
    Path.expand("../examples/adoption_demo/e2e/support/admin-polish.js", __DIR__),
    Path.expand("../brandbook/src/admin-gallery-check.mjs", __DIR__)
  ]

  test "every harness that programmatically drives :focus-visible first enters keyboard modality" do
    for path <- @harnesses do
      src = File.read!(path)
      # Any file that calls focus({focusVisible: true}) MUST also press Tab first —
      # headless Chromium only paints :focus-visible after a real keyboard interaction
      # (the 2026-06-26 "outline 0px != 2px" flake; commit 7c5c5c5 / PR #39).
      if src =~ "focusVisible: true" do
        assert src =~ ~r/keyboard\.press\(["']Tab["']\)/,
               "#{Path.basename(path)} drives :focus-visible without a keyboard.press('Tab') first"
      end
    end
  end
end
```

**B2 — a tiny unit test of the modality helper (only feasible AFTER dedupe).** If §3's dedupe lands, the Tab-first logic becomes one exported function (e.g. `enterKeyboardModality(page)` / `focusVisibly(locator)`). That function is then unit-testable in the Playwright suite against a 1-control fixture page: press → programmatic focus → assert `el.matches(':focus-visible')` is `true`, and the pointer branch is `false`. **Lane:** `adoption-demo-e2e` (push:main, see note). Without dedupe, B2 is not worth two near-identical copies; B1 carries the lock.

> **Lane note:** the live `:focus-visible` assertions run in `adoption-demo-e2e` + `brandbook-tokens`. `adoption-demo-e2e` is `if: != pull_request` (push:main only), but `brandbook-tokens` runs the gallery check (`admin-gallery-check.mjs`) on **all triggers incl. PR** (`ci.yml:1135-1150`, gated only by `github.repository`). So the gallery copy's behavior IS PR-gated; the adoption-demo copy's is main-gated with `adoption-demo-unit` as its PR proxy. **B1 (the text guard) restores a PR-merge-blocking signal for BOTH copies** regardless of where the browser runs — which is the point.

### Lock C — test/ `.planning/`-path hygiene (defect 3)

**The cheapest, most durable lock in the whole milestone.** A meta-test that runs the memory's exact grep as an ExUnit assertion (merge-blocking, ~free):

```elixir
defmodule Rindle.PlanningPathHygieneTest do
  @moduledoc """
  Regression lock (2026-06-26 / PR #45): NO test in test/ may read a `.planning/`
  path at runtime. `.planning/` dirs MOVE/DISAPPEAR when a milestone is archived
  (gsd-cleanup), so any File.read!/exists?/Path.expand over them breaks the
  merge-blocking suite for a non-shipped reason. Keep the decoupling permanent.
  Docstring "see .planning/..." comments are fine — only RUNTIME path access is banned.
  """
  use ExUnit.Case, async: true
  @test_root Path.expand("..", __DIR__)

  test "no test reads a .planning/ path at runtime" do
    offenders =
      Path.wildcard(Path.join(@test_root, "test/**/*.exs"))
      |> Enum.flat_map(fn file ->
        File.read!(file)
        |> String.split("\n")
        |> Enum.with_index(1)
        # Mirrors the memory's grep: File.read!/File.exists?/Path.expand touching .planning
        |> Enum.filter(fn {line, _} ->
          line =~ ~r/(File\.(read!|exists\?)|Path\.expand)[^)]*\.planning/
        end)
        |> Enum.map(fn {line, n} -> "#{Path.relative_to(file, @test_root)}:#{n}: #{String.trim(line)}" end)
      end)

    assert offenders == [],
           "test/ must not read .planning/ paths (they move on archive):\n" <>
             Enum.join(offenders, "\n")
  end
end
```

**Lane:** `quality` (merge-blocking, all triggers). **Cost:** globs `test/**/*.exs` once, pure string scan — sub-second. **This is a strict superset of the install_smoke decoupling**: it guards the whole `test/` tree, not just install_smoke, so it can never silently re-couple anywhere.

## 3. DRY / root-cause note — the two `:focus-visible` copies

**Finding:** the keyboard-modality fix lives in **three call sites across two files**:
- `admin-polish.js:428` (`assertFocusVisibleTokens`) and `:927` (`assertFocusVisibleVsPointer`)
- `admin-gallery-check.mjs:173` (`assertFocusVisibleTokens`, a **separate, hand-duplicated** implementation)

The repo *intentionally* keeps per-script copies of WCAG math (`admin-polish.js:139-167` ports `luminance`/`contrastRatio`/`parseColor` "matching the repo convention of per-script copies rather than cross-package imports" — `:12-14`). But the WCAG copies are **pure, side-effect-free, and already locked by the contrast gates**. The Tab-first modality is different: it is a **stateful browser-quirk workaround** whose correctness is invisible without a keyboard interaction, and a contributor editing one copy (or adding a third focus check) will not know the other exists. **That is the exact footgun that produced the flake in two places at once.**

**Recommendation: DEDUPE the modality logic, keep WCAG copies as-is.**
- `admin-gallery-check.mjs` *already* imports from `admin-polish.js` via `adoptionRequire` (`:25-27` pulls `assertConsistentRhythm`/`assertNoHorizontalScroll`). So a cross-file import path is **already established and load-bearing** — no new convention needed.
- Export a single helper from `admin-polish.js`, e.g. `focusVisibly(page, locator)` (presses `Tab`, then `locator.evaluate(el => el.focus({focusVisible:true}))`), and have all three call sites + the gallery use it.
- This collapses three modality copies into one, makes the assumption documentable in one place, and **enables Lock B2** (a real unit test of the helper).

If the maintainer rejects dedupe (preserving the strict per-script-copy convention even for stateful helpers), fall back to **Lock B1 dual-assert** as the permanent lock — it is cheap and sufficient, just less elegant than removing the duplication. **Locked default: dedupe + B1.** (B1 stays even post-dedupe: it guards against a *future* fourth copy that forgets to import the helper.)

## 5. Idiomatic-for-Elixir/CI analysis — meta-test vs shell vs Playwright

**Which medium for which lock (least-surprise, cheap-to-run):**

| Lock medium | Use it when the asserted fact is… | Rindle precedent | This milestone |
|-------------|-----------------------------------|------------------|----------------|
| **ExUnit meta-test** (`File.read!` shipped artifact + `=~`/AST) | a **static text/structure fact** about a shipped script, workflow, or source file | `ci_lane_split_test.exs`, `ci_cache_hygiene_test.exs`, `package_metadata_test.exs`, `async_safety_guard_test.exs` | **Locks A1, B1, C; Area-1 topology proof** — the default medium |
| **Shell assert in the script itself** | a **runtime precondition** an adopter/CI must satisfy before proceeding | `assert_av_hygiene.sh`, `assert_version_match.sh`, `install_smoke.sh:31-34` self-install | **A2's cold-path purge** (a CI *step*, not a script assert — adopter self-heal must stay silent) |
| **Playwright assertion** | a **rendered/browser-quirk behavior** only observable in a real Chromium | `admin-polish.js`, `admin-gallery-check.mjs` | **B2** (post-dedupe modality unit test) — the only place a browser is required |
| **Deterministic repro loop test** | a **non-deterministic race** that must be reproduced then proven absent | (new for v1.21) | **Area-2 epipe** |

**Ecosystem norm + house rule:** Elixir projects overwhelmingly favor **meta-tests in the default suite** over bash CI-step asserts for "this fact about our config must hold" — because they run locally (`mix ci`), produce a normal test failure with a clear message, and don't depend on a specific CI provider. Rindle has fully internalized this (the entire `test/install_smoke/` family). **Prefer a meta-test over a shell `grep` in a CI step for every static fact;** reserve shell asserts for genuine runtime preconditions. The `.planning/`-hygiene grep (Lock C) is therefore an **ExUnit test, not a CI `grep` step** — same coverage, runs in `mix ci`, fails with a file:line offender list.

## 6. Peer-lib lessons — regression-lock discipline (right-to-emulate + footguns)

How mature OSS locks "we fixed this flake so it can't regress":

**Right to emulate:**
- **Regression test naming/tagging.** Tag the deterministic repro tests `@tag :regression` (or `@moduletag`) so they're greppable and can be run as a focused suite (`mix test --only regression`). Phoenix/Ecto/Oban all name regression tests after the issue they close. → **Adopt for Area-2 epipe.**
- **Lock the FIX's precondition, not just its presence.** A guard that only checks "the fix text exists" but runs in the same warm environment that hid the bug is theater. Exercise the failing condition (Lock A2 purges the archive; the epipe loop forces the race). → **Adopt.**
- **CHANGELOG / commit discipline.** Each fixed flake gets a `fix:`/`ci:` commit referencing the symptom (this repo already does: `7c5c5c5 ci: stabilize admin-polish :focus-visible gate`). Keeps the archaeology cheap. → **Continue.**
- **One required aggregate check, lanes feed it.** Rindle's `CI Summary` (`needs:` all, skipped==pass) is best-practice; new locks just need to land in a job already in `ci-summary.needs` (`quality` is). → **Place every v1.21 lock in `quality`.**
- **Cold-cache CI matrix for cache-sensitive paths.** Mature release pipelines run a periodic cold-cache leg so "works on warm cache only" bugs surface. Rindle's nightly `compat-matrix` already gets cold per-cell builds. → **Lock A2 brings the cold-path proof onto the PR for the phx.new case specifically.**

**Footguns to avoid (called out so requirements dodge them):**
1. **Warm-cache regression test** — reproduces the bug it guards. → A2 purges the precondition.
2. **`.planning/`-coupled meta-test** — breaks on milestone archive for a non-shipped reason (the literal `install_smoke-archive-path` memory; PR #45). → Lock C asserts SHIPPED artifacts only and is itself a guard against re-coupling.
3. **push:main-only / nightly-only lock** — 1-merge MTTD, flake reaches `main` before detection (`ci_lanes_only_on_main` memory). → every v1.21 lock is in `quality` (PR-gating).
4. **Mutable-tag assertion** — asserting a `@vX` action tag that bumps under you (`ci_lane_split_test.exs:9-15` explicitly avoids this). → assert only stable topology/text facts.
5. **Vacuous pass** — a guard that matches nothing and "passes." → Lock B1 keys off the *presence* of `focusVisible: true` so a file that drops the feature doesn't silently exempt itself; Lock C globs and would fail if the glob matched zero files (mirror `async_safety_guard_test.exs`'s `assert files != []`).
6. **Masking instead of fixing** (SEED-004 option c) — tagging+serializing epipe tests hides the race. → pair the repro lock with the actual drain/await fix (option b), and remove the redundant run (option a).

## 7. Design-pillar pass

- **Determinism** — every lock is a pure string/AST scan or a bounded-loop repro; no network, no time, no order dependence. The epipe proof converts a non-deterministic race into a deterministic loop assertion + a structural fix. A2 makes the cold path deterministic (always purged) rather than cache-dependent.
- **Maintainability (cheap locks)** — all locks reuse the existing `test/install_smoke/` meta-test family (one `File.read!` + `=~`), run in the already-required `quality` lane at ~0 added wall-clock, and produce file:line offender messages. No new CI jobs, no new required checks (the `CI Summary` aggregate is unchanged). Dedupe (§3) *reduces* code.
- **Parity (cold == warm)** — A2 deletes the warm-cache precondition so PR == cold runner for the phx.new path. The epipe + coverage-single-run work shrinks the second-run divergence to zero. B-lane note documents PR-vs-main parity for the two focus copies and restores a PR signal via B1.
- **Least-surprise** — meta-tests fail locally under `mix ci` exactly as in CI, with the same message; no provider-specific bash. Adopter self-heal in `install_smoke.sh` stays SILENT (correct UX); the loud cold-path exercise lives in CI only.
