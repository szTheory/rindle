<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions cache → build | a restored cache key can serve artifacts built under a different toolchain if the key under-specifies its dimensions |
| third-party action → runner | `erlef/setup-beam@v1` and `actions/cache@v4` execute on the runner with repo permissions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-104-01 | Tampering | deps/_build cache key (setup-elixir) | mitigate | key carries OS+arch+resolved-OTP+resolved-Elixir+MIX_ENV+lock-hash+v1 (D-05); restore-keys fall back across lock hashes only, never across toolchains (D-05) — a poisoned/incompatible cache cannot be restored across dimensions |
| T-104-02 | Tampering | cache namespace collision (optional-deps) | mitigate | cache-prefix keeps the no-optional tree in a separate namespace (D-06) so a no-optional build can never restore a full-deps cache |
| T-104-SC | Tampering | npm/pip/cargo installs | accept | this plan adds NO package-manager installs; composites reference in-repo `uses: ./…` paths only. Third-party action SHA-pinning + dependabot/mix_audit is explicitly Phase 107 (HARD), not pulled forward (D-16) |
</threat_model>
