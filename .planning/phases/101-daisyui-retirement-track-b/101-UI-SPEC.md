## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | `--ck-1` / 4px | Icon gaps, inline icon/body spacing |
| sm | `--ck-2` / 8px | Compact control padding, tight alert rows, compact alert interior spacing |
| md | `--ck-4` / 16px | Default alert padding, default element gaps, default alert interior spacing |
| lg | `--ck-5` / 24px | Section padding, alert body rhythm where needed |
| xl | `--ck-6` / 32px | Layout gaps only when matching existing Cohort rhythm |
| 2xl | `--ck-7` / 48px | Major section breaks; no new use expected in Phase 101 |
| 3xl | `--ck-8` / 64px | Page-level/footer rhythm; no new alert use |

Legacy token guard: `--ck-3` exists in the Cohort stylesheet from earlier phases, but it is not part of the Phase 101 declared spacing scale and must not be used for new alert or layout padding.

Exceptions: `44px` minimum interactive target for `.ck-alert__dismiss`; `3px` left accent border for `.ck-alert`; `2px` focus outline plus `2px` offset; `1px` borders; existing radius tokens `--ck-radius` and `--ck-radius-sm`. These are not layout spacing and must not become ad hoc margins/padding.

**Phase 101 layout rule:** Remove the `Layouts.app` Tailwind wrapper (`px-4 py-8 sm:px-6 lg:px-8`, `mx-auto max-w-3xl`, `space-y-4`). Routed pages already get width and responsive padding from `ck_page/1` and `.ck__wrap` (`--ck-maxw` 64rem plus clamp padding). Do not replace the wrapper with another utility-style container.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / auxiliary | `--ck-step--1` / 13px | 700 | 1.2 |
| Body / alert message | `--ck-step-0` / 16px | 400 | 1.55 |
| Alert title / compact heading | `--ck-step-1` / 19px | 700 | 1.2 |
| Existing page display | `--ck-step-3` / 29.6px | 700 | 1.1 |

Typography contract:

- New `.ck-flash` and `.ck-alert` CSS may use only the first three rows above.
- Do not introduce new font sizes, negative letter-spacing, all-caps alert titles, or viewport-scaled text in this phase.
- Routine flash messages should be title-less single-line body text. Use a title only when the domain genuinely supplies one.
- Error helper text in `CoreComponents.error/1` must align with `.ck-error`: 13px, 700, icon plus message, never color alone.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `--ck-bg` `#f7f8f6`; dark `--ck-bg` `#0b1411` | Page background and Cohort shell ground |
| Secondary (30%) | Light `--ck-surface` `#ffffff`; dark `--ck-surface` `#111d18`; plus `--ck-surface-2`, `--ck-border`, `--ck-shadow-*` | Alert surface, page surfaces, borders, elevation |
| Accent (10%) | `--ck-brand`, `--ck-brand-strong`, `--ck-focus`, `--ck-info` | Existing primary actions, links, focus ring, info alert left border/icon |
| Destructive | `--ck-quarantine` | Error alert left border/icon and error helper only; no destructive user action is added in this phase |

Accent reserved for: existing `.ck-btn--primary`, links/nav active states, `:focus-visible` rings, `.ck-alert--info` left border and icon, `.ck-alert--error` left border and icon via the destructive token. Do not use accent as a full alert background, page wash, decorative glow, or generic hover color.

Color contract:

- `.ck-alert` uses `--ck-surface` ground, `--ck-border`, `--ck-radius`, `--ck-shadow-lg`, and a `3px` left-border accent via a local `--_accent`.
- `.ck-alert--info` sets `--_accent: var(--ck-info)`.
- `.ck-alert--error` sets `--_accent: var(--ck-quarantine)`.
- Do not add `--ck-info-surface`, `--ck-quarantine-surface`, or any new token value in this phase.
- New CSS must be token-backed for color. No hex, rgb, named colors, Tailwind color classes, daisyUI classes, mask URLs, or data-url icon styles.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Close notification` |
| Empty state heading | none. When no Phoenix flash exists, render no flash DOM |
| Empty state body | none. Do not add placeholder notification copy |
| Error state | Pattern: `{failure} - {next step}`. Preferred upload-domain example: `Upload failed - resume from the last chunk` |
| Destructive confirmation | none. Phase 101 adds no user-visible destructive action; deleting `default.css` is implementation teardown, not UI copy |

Copy voice:

- Developer-adopter posture: terse, factual, no "please", "oops", "sorry", exclamation, emoji, or ornamental marketing copy.
- Info flash confirms a completed operation, for example `Upload complete`.
- Error flash states what failed and, when available, how to continue.
- Dismiss label is exactly `Close notification`, capitalized as sentence case in `aria-label`.

## Phase-Specific Visual Contract

### Flash / Alert Primitive

- Add only `.ck-flash`, `.ck-alert`, `.ck-alert--info`, `.ck-alert--error`, `.ck-alert__icon`, `.ck-alert__body`, `.ck-alert__title`, `.ck-alert__msg`, and `.ck-alert__dismiss` unless an adjacent BEM child is needed for markup clarity.
- `CoreComponents.flash/1` must render under `class="ck ck-flash"` so the Cohort focus ring and reduced-motion clamp apply.
- `.ck-flash` is a fixed top-end notification container above `.ck-nav` (`z-index: 50`; nav is 20). It must stay within the viewport on mobile using token-based side offsets and a responsive width cap.
- `.ck-alert` is a compact surface, not a modal, drawer, or card. It uses icon + message + optional title + close button, with no nested card.
- The close button is a real `<button type="button">`, keeps the existing Phoenix JS click-to-dismiss behavior, has a `44px` hit target, and does not cause layout shift on hover/focus.
- Use the existing `ck-rise` animation only inside `@media (prefers-reduced-motion: no-preference)`. Do not add decorative animation, timers, or delayed error/failure updates.

### Icons

- Phase 101 is an explicit exception to the nested Phoenix generator note that normally points to `<.icon>`. The existing `<.icon>` renders glyphless Heroicon class spans that depend on `default.css`; those classes must be retired.
- Inline the information, exclamation, and close glyphs directly as SVG in the flash/error helper path. SVGs inherit `currentColor`, use stable viewBox geometry, and are `aria-hidden="true"` because visible text carries the accessible name.
- Do not create `hero-*` classes in `cohort.css`. Do not add `mask`, `-webkit-mask`, or `url(data:image/svg+xml...)` icon rules.

### Layout

- `Layouts.app/1` remains the only place that calls `<.flash_group>`, matching Phoenix 1.8 convention.
- `Layouts.app/1` renders `cohort_nav`, a bare `<main>` with the inner slot, `cohort_footer`, and `flash_group` as direct app-level children.
- Do not wrap all pages in a new `.ck` root at the layout level. Routed inner pages already own their `.ck`/`data-ck-root` through `ck_page/1`; flash gets its own `.ck ck-flash` wrapper because it lives outside page roots.
- `root.html.heex` keeps `app.css` and `cohort.css`. Remove only the `default.css` link.

## Interaction And Accessibility Contract

- `:info` flash uses `role="status"` and `aria-live="polite"`.
- `:error` flash uses `role="alert"` and `aria-live="assertive"`.
- `flash_group/1` may keep its existing outer `aria-live="polite"` wrapper, but the per-kind alert/status semantics above are authoritative.
- Flash never steals focus and never auto-dismisses.
- Dismiss is pointer and keyboard operable, visible at rest, and uses the Cohort `:focus-visible` ring.
- State is never communicated by color alone: info/error alerts use icon + text + left-border accent.
- Light, dark, and system theme behavior must continue to resolve through existing `:root`, `[data-theme="light"]`, `[data-theme="dark"]`, and `prefers-color-scheme` token blocks.
- No UI text may depend on hover-only disclosure.
