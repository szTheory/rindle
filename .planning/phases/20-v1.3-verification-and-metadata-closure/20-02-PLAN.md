<!-- Key contract — extracted from lib/rindle/upload/broker.ex (read it once before executing) -->

`Rindle.Upload.Broker.sign_url(session_id) :: {:ok, %{session: signed_session, presigned: presigned}} | {:error, term()}`

Where:
- `signed_session` is a `%MediaUploadSession{}` struct with `:state == "signed"`, `:id`, `:upload_key`, `:asset_id` populated from a real Repo write.
- `presigned` is a map with `:url`, `:method` (default "PUT"), `:headers` (default %{}).

The broker is the single source of truth for the asset_id-session-storage_key relationship. The fabricated UUID approach the LiveView previously used produced an asset_id with no row, breaking adopters who used the meta downstream.

Working-tree patch summary (already on disk — confirmed via `git diff --stat`):
- `lib/rindle/live_view.ex` (+13 lines net): adds `alias Rindle.Upload.Broker`; refactors `handle_initiate_upload/3` to call `Broker.sign_url(session.id)` and unwrap `signed_session`/`presigned`; meta uses `signed_session.id` and `signed_session.asset_id` instead of `session.id` + fabricated UUID.
- `test/rindle/live_view_test.exs` (+98 lines net): adds `Phoenix.LiveView.{UploadConfig, UploadEntry}` + `Rindle.Domain.{MediaAsset, MediaUploadSession}` aliases; replaces the placeholder `external signer calls initiate_upload and presigned_put` test with a real broker-backed round-trip; replaces the placeholder `module is defined and has consume_uploaded_entries/3` test with a Mox `:head` expectation that proves session state transitions to `"completed"` and asset state transitions to `"validating"` with real `byte_size`/`content_type` data.
