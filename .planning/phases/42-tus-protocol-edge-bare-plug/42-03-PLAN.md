<objective>
Build the write/complete/terminate half of `Rindle.Upload.TusPlug`: the `PATCH`
offset gate (`409` without reading the body), the `Content-Type` gate (`415`), the
streaming `read_body` append loop with a per-PATCH ceiling (`413`), completion
convergence (atomic rename → the UNCHANGED `verify_completion/2` → a `ready`
`MediaAsset`), `DELETE` termination, and the full tus-js-client-shaped resume
contract flow (POST → HEAD → PATCH-partial → simulated drop → HEAD → PATCH-resume
→ completion). The `409` and the resume flow are the contract test's spine.

Purpose: Land the PATCH hot path and the completion convergence — the protocol's
load-bearing core and the parts a hand-rolled Plug most plausibly gets wrong —
with zero new completion vocabulary (D-08) and no buffering of bodies (D-07).
Output: PATCH/DELETE handlers in `tus_plug.ex`, the full contract flow test, and
the Local backing→promotion test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tus client → PATCH | untrusted streaming body + `Upload-Offset`/`Content-Type` headers cross here; body must be bounded and offset-gated |
| PATCH → tmp file → final key | server-issued `session_id` path; atomic rename into the storage root |
| completion → verify_completion/2 | only a server-verified `session_id` crosses; head-based re-sniff is the trust point |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-DOS | DoS | PATCH streaming body | mitigate | `read_body` `read_length: 1_048_576` + per-PATCH ceiling from `max_size` → `413`; never buffer the whole body (D-07). Test: 413 on over-ceiling. |
| T-42-RACE | Tampering | offset gate | mitigate | Strict `Upload-Offset == last_known_offset` integer gate → `409` WITHOUT reading the body; offset unchanged; 1:1 session↔tmp file. Test asserts body not consumed. |
| T-42-META | Tampering | `Upload-Metadata` (carried from POST) | mitigate | Opaque/untrusted; never trusted for path/MIME; re-sniffed at `verify_completion` head trust lane (invariants 1, 10). |
| T-42-XFS | Tampering / Integrity | completion atomic rename | mitigate | tmp + storage root same filesystem; `File.rename` atomic; `:exdev` = misconfig error, not a silent copy+delete (Pitfall 5). Backing test asserts final size. |
| T-42-FSM | Tampering | completion FSM transition | mitigate | Keep session in `"signed"`/`"uploading"` so `-> "verifying"` is legal; never `"resuming"` at completion (Pitfall 7). |
| T-42-LEAK | Information Disclosure | signed URL on PATCH/DELETE | mitigate | URL lives only in redacted `session_uri`; PATCH/DELETE handlers never `Logger`/`inspect` the raw URL (invariant 14). |
| T-42-FORGE/REPLAY | Spoofing / Elevation | token verify on PATCH/DELETE | mitigate | PATCH/DELETE dispatch verifies the HMAC token FIRST (Plan 02 `verify_token/2`); tampered → 404, expired token → 401, expired session → 410; never 200. |
</threat_model>
