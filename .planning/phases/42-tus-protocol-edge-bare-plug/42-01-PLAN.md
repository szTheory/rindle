<objective>
Build the entire non-HTTP foundation the tus Plug stands on: the `:tus_upload`
capability atom (registered + honestly advertised), the single additive
`resumable_protocol` migration + schema field, the `Broker.initiate_tus_upload/2`
session-creation entrypoint, and the Local tmp-append + atomic-rename backing
helper. This is everything `TusPlug` (Plans 02/03) consumes — built first so the
Plug receives concrete contracts, not a scavenger hunt.

Purpose: De-risk the seams before any HTTP mechanics. The Plug is ~95% glue over
these primitives (D-01, D-09, D-10, D-11); they must exist and be tested before
the edge is wired.
Output: Capability registration, migration, schema field, broker entrypoint,
Local backing helper, and a backing/capability unit test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| broker → DB | session row persisted; `resumable_protocol`/`upload_strategy`/`state` are server-set, never client-influenced in this plan |
| Local helper → filesystem | tmp part path built from server-issued `session_id` (UUID); atomic rename into the storage root |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-PATH | Tampering | `Local.tus_part_path/2` | mitigate | Path built only from server-issued UUID `session_id` under `root/1`; no client input reaches the path. The HMAC-token gate that produces `session_id` lands in Plan 02; this plan keeps the path-construction structurally traversal-proof. |
| T-42-XFS | Tampering / Integrity | `Local.tus_complete` atomic rename | mitigate | tmp dir + storage root both under `root/1` (same filesystem) → `File.rename/2` is atomic; `:exdev` surfaced as an error (misconfig), never a silent copy+delete fallback (Pitfall 5). Test asserts final size == appended bytes. |
| T-42-FSM | Tampering | broker session state | mitigate | Persist tus sessions in `"signed"` so the completion edge `signed → verifying` is legal; never park in `"resuming"` (`resuming → verifying` is illegal, Pitfall 7 / FSM allowlist verified). |
| T-42-DOWNGRADE | Elevation | capability gate | mitigate | `initiate_tus_upload/2` gates on `require_upload(adapter, :tus_upload)`; an adapter lacking the capability cannot create a tus session — no silent downgrade (D-09). |
</threat_model>
