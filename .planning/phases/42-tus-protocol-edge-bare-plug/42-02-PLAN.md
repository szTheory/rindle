<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| third-party tus client тЖТ TusPlug | untrusted HTTP: method, headers (`Upload-Length`, `Upload-Metadata`, `Upload-Offset`), and the path-segment token all cross here |
| TusPlug тЖТ Broker/DB | only a server-verified `session_id` from the HMAC payload crosses (never raw client input) |
| TusPlug тЖТ logs/telemetry/inspect | the signed tus URL must NOT cross this boundary except via the redacted `session_uri` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-FORGE | Spoofing / Elevation | token verify on HEAD | mitigate | `Plug.Crypto.verify` HMAC against `secret_key_base` (never hand-rolled); forged/tampered token тЖТ `404`, never `200`. Test: tampered/forged тЖТ non-200. |
| T-42-REPLAY | Elevation | `exp` check + `expires_at` | mitigate | Manual `exp >= now` check in payload тЖТ `401`; session `expires_at` past тЖТ `410 Gone`. Tests assert both, never 200. |
| T-42-LEAK | Information Disclosure | signed tus URL | mitigate | Stored only in `session_uri` (redacted `Inspect`, media_upload_session.ex:104-113); never `Logger`/telemetry/`inspect`. Test asserts inspect redaction. |
| T-42-META | Tampering | `Upload-Metadata` on POST | mitigate | Treated opaque/untrusted; NOT parsed for filename/MIME/path; re-sniffed at `verify_completion` (Plan 03). Source assertion. |
| T-42-DOWNGRADE | Elevation | `init/1` capability gate | mitigate | `init/1` raises `ArgumentError` when adapter lacks `:tus_upload` тАФ deploy-time failure, no silent downgrade (D-09). Test: init-raise. |
| T-42-PATH | Tampering | token тЖТ session_id тЖТ (future) tmp path | mitigate | `session_id` comes ONLY from the verified HMAC payload (server UUID); `within_root?/2` guard applied defensively; tampered token тЖТ 404 before any path is built. |
| T-42-DOS-POST | DoS | POST `Upload-Length` | mitigate | `Upload-Length > max_size` тЖТ `413` at create time (bounds the per-session sink before any bytes flow). |
