## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip marks, focus offset |
| sm | 8px | Compact inline padding, nav item rhythm |
| md | 16px | Default control spacing, table cells, filter groups, stacked field gaps |
| lg | 24px | Section padding, panel internals, detail groups |
| xl | 32px | Shell gutters, page header spacing, primary layout gaps |
| 2xl | 48px | Major surface section breaks and page top/bottom padding |
| 3xl | 64px | Rare page-level separation only; avoid inside dense read surfaces |

Exceptions:

- `12px` may be used only via `--rindle-space-3` for dense table cells, chip padding, compact nav rows, or detail metadata rows where `8px` is too tight and `16px` is too loose.
- Interactive targets must be at least `44px` high/wide for nav items, filter controls, buttons, table row actions, theme controls, drawer/dialog controls, and toast dismiss controls.
- Focus rings use `2px` width and `2px` offset from brand tokens.

Source: `brandbook/tokens/tokens.json`; `guides/ui_principles.md`; `guides/rindle_admin_css.md`; `88-UI-SPEC.md`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 600 | 1.45 |
| Body | 17px | 400 | 1.6 |
| Heading | 22px | 600 | 1.25 |
| Display | 30px | 600 | 1.15 |

Typography contract:

- Use exactly two weights in console UI: `400` regular and `600` semibold.
- Use sentence case for surface titles, nav items, filter labels, table headings, empty states, error states, and toast messages.
- Do not use hero-scale `44px` or `64px` typography in Phase 89; this is an operational console, not a landing page.
- Use mono text only for asset IDs, session IDs, variant names, route names, env vars, file paths, commands, and redacted provider/session identifiers.
- Tables and detail pages must remain scannable: labels at `14px`, body cells at `17px`, stable line-height, and no squeezed all-caps headers except tiny metadata labels.

Source: `brandbook/tokens/tokens.json`; `guides/ui_principles.md`; `88-UI-SPEC.md`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F7F4EA` / `--rindle-surface` | Main page background and console shell background |
| Secondary (30%) | `#FBFEFC` / `--rindle-surface-raised` | Nav/sidebar surface, panels, tables, detail sections, drawers, toasts, empty states |
| Accent (10%) | `#123A35` / `--rindle-brand` | Primary CTA, active nav indicator, selected theme segment, focus ring on light surfaces |
| Destructive | `#C83232` / `--rindle-status-danger` | Phase 89 read-only destructive previews/labels only; executable destructive actions belong to Phase 90 |

Accent reserved for: active nav marker, primary investigation CTA, selected filter/theme state, focus ring on light surfaces, sparse brand marks, and one primary focus point per surface. Do not use accent for every link, badge, table border, chart mark, or decorative highlight.

Additional semantic color rules:

- Dark theme uses `#0E1316` dominant and `#161E23` secondary via `data-theme="dark"` or `data-theme="auto"` plus `prefers-color-scheme`.
- `#32D08C` / `--rindle-accent` is not text-safe on light surfaces; reserve it for dark-surface accent text, readiness graphics, dark focus rings, and small non-text highlights.
- Status colors must use token-gated foreground/background pairs and visible labels or inline marks. Never rely on color alone.
- `processing #6D5DD3` is frozen; do not lighten it because its AA contrast margin is intentionally narrow.
- Phase 89 surfaces must render ready, processing, warning, danger, quarantine, info, empty, loading, focus, and live-updated states in both light and dark themes.

Source: `brandbook/tokens/tokens.json`; `guides/rindle_admin_css.md`; `guides/admin_design_system.md`; `88-UI-SPEC.md`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Inspect assets |
| Empty state heading | No records match this view |
| Empty state body | Adjust the filters or review Runtime/Doctor to confirm Rindle is receiving lifecycle events. |
| Error state | Rindle Admin could not load this surface. Review the runtime checks, then retry after the missing source is available. |
| Destructive confirmation | None in Phase 89: Actions is read-only and may link toward future owner erasure, repair, regeneration, and quarantine flows without executing them. |

Copywriting contract:

- Surface titles must be exactly `Home/Status`, `Assets`, `Upload Sessions`, `Variants/Jobs`, `Runtime/Doctor`, and `Actions`.
- Use direct operator verbs: `Inspect`, `Filter`, `Review`, `Open`, `Refresh`, `Retry`, `Copy`, and `View`.
- Empty states must name the missing data and the next useful diagnostic action.
- Error states must name the failed surface or missing source and point the operator to `Runtime/Doctor` or retry.
- Live-update copy must be factual and restrained: `Updated just now`, `Waiting for lifecycle events`, or `Refresh failed; retry`.
- Redacted values must say `Redacted by Rindle Admin` or `Provider identifier redacted`; never expose upload `session_uri` values or raw provider-internal IDs.

Source: `89-CONTEXT.md` D-89-12 through D-89-19; `guides/admin_console_ia.md`; `guides/ui_principles.md`; `REQUIREMENTS.md` ADMIN-03 and ADMIN-05.

## Interaction Contract

Phase 89 implements the read-only console experience that Phase 90 actions and Phase 92 E2E will build on.

Visual hierarchy contract:

- The console is a dense operational tool. Start on health, blocked work, and next investigation routes; avoid decorative dashboard cards, marketing hero layouts, or analytics sprawl.
- Every page must use the `rindle-admin-shell` navigation, active surface state, service identity, theme picker, and stable selector attributes.
- Read surfaces must use `Rindle.Admin.Queries` as rendered truth. PubSub events trigger query refresh; they are not complete UI data.
- Loading states must use skeletons only for real pending work and must keep stable dimensions to avoid layout shift.

| Area | Contract |
|------|----------|
| Shell/navigation | Use one persistent nav with the six locked surfaces. The active page must expose `aria-current="page"` and a stable `data-rindle-admin-surface` value. |
| Home/Status | Show runtime summary, doctor summary, counts for assets/upload sessions/variants/jobs, recommendations, and links to the relevant investigation surface. Do not place destructive controls here. |
| Assets | Provide lifecycle-state filters, profile/kind filters when available, a scannable asset table, and asset detail with attachments, variant summary, and state/timeline context. |
| Upload Sessions | Show state, asset link, strategy/protocol summary, expiration, failure reason, and cleanup guidance. Redact upload `session_uri` values. |
| Variants/Jobs | Show variant state, stale/missing/failed/cancelled buckets, safe job correlation where available, and the recommended repair lane without executing repair. |
| Runtime/Doctor | Present doctor checks and runtime status as read-only diagnostics, with failed or missing prerequisites named plainly. |
| Actions | Phase 89 Actions is a read-only directory of future operations. It may explain owner erasure, batch erasure, regeneration, quarantine review, and lifecycle repair, but all execution controls remain absent or disabled until Phase 90. |
| Filters | Filters must be keyboard reachable, preserve selected state visibly, and update result counts without changing the overall layout footprint. |
| Detail navigation | Asset/session/variant detail must be reachable from list rows and row actions. Row actions use explicit labels such as `Inspect` or `Review`. |
| Live updates | Subscribe only to visible relevant topics (`:asset`, `:variant`, `:upload_session`) and refresh through `Rindle.Admin.Queries`. Show unobtrusive update status; do not animate delayed failure/destructive state. |
| Theme | Theme picker writes `data-theme="light\|dark\|auto"`; `auto` follows `prefers-color-scheme`. Both light and dark states must preserve chip, table, focus, empty, and error readability. |
| Status chips | Include visible label plus icon or equivalent non-color mark; use `ready`, `processing`, `warning`, `danger`, `quarantine`, and `info` token pairs. |
| Buttons/links | Use primary sparingly for the main investigation CTA on a surface. Secondary/quiet buttons handle filters, row actions, refresh, copy, and retry. |
| Accessibility | All controls keyboard reachable; no UI text depends on hover-only disclosure; focus-visible uses brand focus tokens; targets are at least `44px`; status is never color-only. |
| Motion | Use token durations only: press `120ms`, popover `160ms`, toast `200ms`, transition `300ms`; respect `prefers-reduced-motion`; no decorative animation or infinite loading animation. |
| Stable selectors | Add stable `data-rindle-admin-*` attributes for root, surface, nav item, filter, row, status chip, empty state, loading state, error state, and live-update indicator. |

Source: `89-CONTEXT.md`; `89-RESEARCH.md`; `guides/admin_console_ia.md`; `guides/admin_design_system.md`; `guides/ui_principles.md`.
