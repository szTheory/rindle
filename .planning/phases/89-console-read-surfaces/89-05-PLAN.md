<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime/doctor data -> browser | Diagnostic findings render to operators. |
| action directory -> operator expectations | Read-only operation categories must not be mistaken for executable flows. |
| PubSub -> Variants/Jobs | Variant and asset events invalidate visible rows. |
| optional LiveView boundary -> default adopters | New LiveViews must compile away without LiveView. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-89-21 | Elevation of Privilege | `ActionsLive` | mitigate | Render disabled/read-only operation entries only; scan for destructive/repair event handlers and facade calls. |
| T-89-22 | Information Disclosure | `VariantsJobsLive` provider/job context | mitigate | Use query-level redaction and render `Provider identifier redacted`; do not render raw provider IDs. |
| T-89-23 | Tampering | PubSub refresh | mitigate | Treat asset/variant events as invalidation and re-query `Rindle.Admin.Queries.variants_jobs/1`. |
| T-89-24 | Repudiation | runtime diagnostics | mitigate | Render doctor/runtime generated-at/check status so operators can distinguish diagnostics from actions. |
| T-89-25 | Denial of Service | LiveView optional deps | mitigate | Guard modules and run no-optional-deps compile after implementation. |
| T-89-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install task is planned in this LiveView plan. |
</threat_model>
