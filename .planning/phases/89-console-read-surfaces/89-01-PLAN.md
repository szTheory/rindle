<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host router -> Rindle macro | Host-controlled scope/pipeline invokes `Rindle.Admin.Router.rindle_admin/2`. |
| host auth -> LiveView on_mount | Host-owned auth/session checks cross into Rindle LiveViews through `:on_mount`. |
| router macro -> static asset plug | Macro exposes library-owned static assets into the host request path. |
| optional dependency boundary -> default adopters | Non-console adopters compile without Phoenix LiveView. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-89-01 | Elevation of Privilege | `Rindle.Admin.Router.rindle_admin/2` | mitigate | Require non-empty `:on_mount` or `auth_guarded?: true` for production mounts; reject `allow_unauthenticated?: true` in production. |
| T-89-02 | Spoofing | Host auth acknowledgement | mitigate | Keep auth ownership in host pipeline/on_mount; Rindle only accepts explicit acknowledgement and does not invent sessions or actors. |
| T-89-03 | Tampering | Static asset route | mitigate | Use `Plug.Static` with `from: {:rindle, "priv/static/rindle_admin"}` and exact `only:` list. |
| T-89-04 | Information Disclosure | CSP/socket options | mitigate | Apply host-provided `:csp_nonce_assign_key`; do not generate Rindle-owned nonces or expose host session data. |
| T-89-05 | Denial of Service | Optional LiveView dependency | mitigate | Guard all Phoenix/LiveView references with `Code.ensure_loaded?` and verify `mix compile --no-optional-deps --warnings-as-errors`. |
| T-89-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install task is planned in this router plan. |
</threat_model>
