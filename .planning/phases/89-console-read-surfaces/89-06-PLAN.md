<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upload lifecycle code -> PubSub | Broker/TusPlug successful state changes emit messages. |
| PubSub payload -> LiveView | LiveViews receive events that may be forged or partial. |
| upload session row -> payload | Sensitive fields must be excluded from event payloads. |
| PubSub topology -> console | Console must reuse existing `Rindle.PubSub` topics only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-89-26 | Information Disclosure | upload-session PubSub payload | mitigate | Payload allowlist excludes `session_uri`, headers, tokens, and provider IDs; tests assert absence. |
| T-89-27 | Tampering | forged PubSub payloads | mitigate | Console tests prove LiveViews treat payloads as invalidation and re-query authoritative data. |
| T-89-28 | Denial of Service | over-broad subscriptions/channels | mitigate | Reuse exact existing topic grammar and `Rindle.PubSub`; no second realtime channel. |
| T-89-29 | Repudiation | lifecycle notifications | mitigate | Event atoms map to successful persisted transitions only, not failed validation attempts. |
| T-89-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install task is planned in this PubSub plan. |
</threat_model>
