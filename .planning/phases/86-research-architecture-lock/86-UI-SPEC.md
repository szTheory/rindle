# Phase 86 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none; Phase 86 locks a manual `rindle-admin` design system, generated from `brandbook/tokens/tokens.json` |
| Preset | not applicable |
| Component library | none; shipped console components use vanilla Phoenix/LiveView markup plus BEM classes |
| Icon library | none for runtime dependency; status/action icons must be inline, self-contained, and paired with text labels |
| Font | `Space Grotesk` for compact headings; `Atkinson Hyperlegible` for UI/body; `JetBrains Mono` for code/IDs |

Source: `86-CONTEXT.md` D-86-03, D-86-11, D-86-13, D-86-15; `86-RESEARCH.md` Standard Stack; `brandbook/tokens/tokens.json`.

Design-system contract:

- The shipped admin console must not depend on host Tailwind, daisyUI, esbuild, or asset-pipeline integration.
- Console CSS must be a vanilla `rindle-admin` layer generated from brand tokens, using BEM selectors and CSS custom properties.
- Cohort may keep Tailwind/daisyUI momentum; the library console must not inherit Cohort's frontend stack.
- Phase 86 itself produces docs/ADRs/UI principles, not production console components.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip dot spacing, focus offset |
| sm | 8px | Compact inline padding, nav item rhythm |
| md | 16px | Default control spacing, table cell rhythm, stacked field gaps |
| lg | 24px | Section padding, panel internals, doc block spacing |
| xl | 32px | Layout gutters, sidebar/header spacing |
| 2xl | 48px | Major section breaks, page top/bottom padding |
| 3xl | 64px | Large docs/principles page separation only |

Exceptions:

- `12px` may be used only where the existing token `--rindle-space-3` is already the best fit for dense table cells, chip padding, or compact nav rows.
- Interactive targets must be at least `44px` high/wide for buttons, icon buttons, theme controls, and destructive confirmation controls.
- Focus rings use `2px` width and `2px` offset from brand tokens.

Source: `brandbook/tokens/tokens.json`; default 8-point contract narrowed for an operational console.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 600 | 1.45 |
| Body | 17px | 400 | 1.6 |
| Heading | 22px | 600 | 1.25 |
| Display | 30px | 600 | 1.15 |

Typography contract:

- Use exactly two weights in console UI: `400` regular and `600` semibold.
- Use sentence case for headings, nav labels, buttons, empty states, and error copy.
- Use display type sparingly for page/surface titles only; do not use hero-scale `44px` or `64px` typography in the admin console.
- Use mono text only for code, IDs, route names, env vars, file paths, and command snippets.
- Prose in `guides/ui_principles.md` and related Phase 86 docs should keep line length between `68ch` and `78ch`.

Source: `brandbook/tokens/tokens.json`; Phase 86 operational-console adaptation.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F7F4EA` / `--rindle-surface` | Main page background, documentation surface, console shell background |
| Secondary (30%) | `#FBFEFC` / `--rindle-surface-raised` | Panels, tables, nav/sidebar surface, drawers, dialogs, empty states |
| Accent (10%) | `#123A35` / `--rindle-brand` | Primary CTA, active nav indicator, focus ring on light surfaces, selected segmented-control state |
| Destructive | `#C83232` / `--rindle-status-danger` | Destructive actions only: owner erasure, batch erasure, quarantine/lifecycle destructive warnings |

Accent reserved for: primary CTA background, active nav marker, selected theme/segmented state, focus ring on light surfaces, and sparse brand marks. Do not use accent for every link, badge, border, chart, or decorative highlight.

Additional semantic color rules:

- `#32D08C` / `--rindle-accent` is not text-safe on light surfaces; reserve it for dark-surface accent text, readiness graphics, dark focus rings, and small non-text highlights.
- Status colors must use token-gated foreground/background pairs and must include visible text labels or icons. Never rely on color alone.
- `processing #6D5DD3` is frozen; do not lighten it because the contrast margin is narrow.
- Dark theme uses `#0E1316` dominant and `#161E23` secondary via `data-theme="dark"` or `data-theme="auto"` plus `prefers-color-scheme`.

Source: `brandbook/tokens/tokens.json`; `86-CONTEXT.md` D-86-12, D-86-13.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Lock architecture |
| Empty state heading | No architecture decisions recorded |
| Empty state body | Create the Phase 86 architecture notes before planning implementation phases. |
| Error state | Architecture lock could not be verified. Review the missing source artifact and rerun the checker. |
| Destructive confirmation | none in Phase 86; later console destructive actions must require typed confirmation plus collateral preview |

Copywriting contract:

- Phase 86 docs must use direct operator language: "Lock", "Review", "Verify", "Mount", "Serve", "Repair", "Erase".
- Avoid marketing copy, dashboard flourish, and vague status language.
- Later destructive flows must name the action and affected owner/assets before enabling confirmation.

Source: `86-CONTEXT.md` D-86-19, D-86-20; `REQUIREMENTS.md` ADMIN-04.

## Interaction Contract

Phase 86 must lock these interaction rules for phases 87-93:

| Area | Contract |
|------|----------|
| Navigation | Use task-first IA: Home/Status, Assets, Upload Sessions, Variants/Jobs, Runtime/Doctor, Actions. Do not create decorative analytics dashboard sprawl. |
| Theme | Theme control is `light`, `dark`, `auto`; implementation uses `data-theme="light\|dark\|auto"` and `prefers-color-scheme`. |
| Motion | Use token durations only: press `120ms`, popover `160ms`, toast `200ms`, transition `300ms`; respect `prefers-reduced-motion`; no decorative animation. |
| Feedback | Async actions show real state immediately. Use loading/disabled states only while work is actually pending. |
| Status | Lifecycle chips include label/icon plus color; ready, processing, warning, danger, quarantine, and info use token pairs. |
| Destructive actions | Owner erasure and batch erasure require collateral preview, typed confirmation, disabled-submit until matched, and post-action receipt. |
| Accessibility | All controls are keyboard reachable, focus-visible uses brand focus tokens, contrast gates reuse `brandbook/src/contrast.mjs` patterns. |
| Self-contained assets | UI must render from library-owned static assets; host apps provide auth/mounting, not styling. |

Source: `86-CONTEXT.md` D-86-08 through D-86-15; `86-RESEARCH.md` Architecture Patterns.
