<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generated gallery -> browser | Static HTML and inline theme behavior execute in a local browser for maintainer review. |
| theme picker -> DOM attribute | Theme controls write the trusted `data-theme` value used by CSS. |
| gallery screenshot tool -> filesystem | Playwright writes PNG review artifacts under `brandbook/admin-gallery/screenshots/`. |
| gallery examples -> future implementation | Gallery markup becomes the reference pattern consumed by Phase 89 and Phase 90. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-88-06 | Spoofing | Theme picker | mitigate | Allowlist `light`, `dark`, and `auto`; browser check asserts exact `data-theme` transitions and forbids `.dark`/`theme-dark` conventions. |
| T-88-07 | Tampering | Component gallery source | mitigate | Gallery is regenerated from allowlisted data and generated CSS; stable selectors and source assertions block missing required components/states. |
| T-88-08 | Information Disclosure | Static gallery examples | mitigate | Use hardcoded non-secret fixture values only; do not render real media metadata, account identifiers, tokens, or environment values. |
| T-88-09 | Denial of Service | Screenshot harness | mitigate | Static `file://` rendering avoids starting the Phoenix app; script closes the browser and writes bounded seven-file output. |
| T-88-10 | Elevation of Privilege | Phase boundary | mitigate | Task explicitly forbids router macro, auth contract, Plug.Static, CSP/socket, and `Rindle.Admin.Queries` implementation. |
| T-88-SC | Tampering | npm installs | accept | No package-manager install task is planned; the script reuses existing Playwright dependencies audited in research. |
</threat_model>
