# Phase 88 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none; Phase 88 implements the manual `rindle-admin` design system generated from `brandbook/tokens/tokens.json` |
| Preset | not applicable |
| Component library | none; shipped console components use vanilla Phoenix/LiveView-ready markup plus BEM classes |
| Icon library | none as a runtime dependency; status/action marks must be inline, self-contained, and paired with visible text labels |
| Font | `Space Grotesk` for compact headings; `Atkinson Hyperlegible` for UI/body; `JetBrains Mono` for code/IDs |

Source: `88-CONTEXT.md` D-88-01 through D-88-04, D-88-09 through D-88-13; `88-RESEARCH.md` Standard Stack; `86-UI-SPEC.md`; `brandbook/tokens/tokens.json`.

Design-system contract:

- Generate a dedicated vanilla `rindle-admin` CSS layer from `brandbook/tokens/tokens.json`.
- Use inspectable BEM selectors, including `.rindle-admin-shell`, `.rindle-admin-nav__item`, `.rindle-admin-table__row`, `.rindle-admin-status-chip`, and state modifiers such as `.rindle-admin-status-chip--processing`.
- Generate `--rindle-` CSS custom properties and component CSS from source files; do not hand-edit generated CSS artifacts.
- Keep the shipped library console independent of host Tailwind, daisyUI, esbuild, asset-pipeline integration, shadcn, Radix, Tailwind UI, daisyUI registries, or other third-party UI registries.
- Cohort may keep its own Tailwind/daisyUI stack, but no Cohort styles may leak into the shipped `rindle-admin` layer.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip marks, focus offset |
| sm | 8px | Compact inline padding, nav item rhythm |
| md | 16px | Default control spacing, table cell rhythm, stacked field gaps |
| lg | 24px | Section padding, panel internals, drawer/dialog padding |
| xl | 32px | Layout gutters, shell/sidebar/header spacing |
| 2xl | 48px | Major gallery section breaks, screen top/bottom padding |
| 3xl | 64px | Gallery page-level separation only |

Exceptions:

- `12px` may be used only via the existing token `--rindle-space-3` for dense table cells, chip padding, or compact nav rows where `8px` is too tight and `16px` is too loose.
- Interactive targets must be at least `44px` high/wide for buttons, icon buttons, theme controls, drawer triggers, row actions, destructive confirmation controls, and toast dismiss controls.
- Focus rings use `2px` width and `2px` offset from brand tokens.

Source: `brandbook/tokens/tokens.json`; `guides/ui_principles.md`; `86-UI-SPEC.md`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 600 | 1.45 |
| Body | 17px | 400 | 1.6 |
| Heading | 22px | 600 | 1.25 |
| Display | 30px | 600 | 1.15 |

Typography contract:

- Use exactly two weights in console UI: `400` regular and `600` semibold.
- Use sentence case for component labels, nav items, table headings, empty states, error states, toasts, and destructive confirmations.
- Use display type only for surface or gallery titles; do not use hero-scale `44px` or `64px` typography in the admin console.
- Use mono text only for code, IDs, route names, env vars, file paths, command snippets, and deterministic selector examples.
- Tables must favor scannability: labels at `14px`, body cells at `17px`, stable line-height, and no squeezed all-caps headers beyond tiny labels.

Source: `brandbook/tokens/tokens.json`; `86-UI-SPEC.md`; `guides/ui_principles.md`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F7F4EA` / `--rindle-surface` | Main page background, gallery background, console shell background |
| Secondary (30%) | `#FBFEFC` / `--rindle-surface-raised` | Panels, tables, nav/sidebar surface, drawers, dialogs, toasts, empty states |
| Accent (10%) | `#123A35` / `--rindle-brand` | Primary CTA, active nav indicator, selected theme segment, focus ring on light surfaces |
| Destructive | `#C83232` / `--rindle-status-danger` | Destructive actions only: owner erasure, batch erasure, destructive warnings, destructive confirmation affordances |

Accent reserved for: primary CTA background, active nav marker, selected theme/segmented state, focus ring on light surfaces, sparse brand marks, and one primary focus point per component example. Do not use accent for every link, badge, table border, chart mark, or decorative highlight.

Additional semantic color rules:

- Dark theme uses `#0E1316` dominant and `#161E23` secondary via `data-theme="dark"` or `data-theme="auto"` plus `prefers-color-scheme`.
- `#32D08C` / `--rindle-accent` is not text-safe on light surfaces; reserve it for dark-surface accent text, readiness graphics, dark focus rings, and small non-text highlights.
- Status colors must use token-gated foreground/background pairs and visible labels or inline marks. Never rely on color alone.
- `processing #6D5DD3` is frozen; do not lighten it because its AA contrast margin is intentionally narrow.
- The gallery and contrast gate must cover ready, processing, warning, danger, quarantine, info, empty, loading, and focus states in light and dark themes.

Source: `brandbook/tokens/tokens.json`; `88-CONTEXT.md` D-88-07, D-88-08, D-88-18, D-88-19; `guides/rindle_admin_css.md`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Review component gallery |
| Empty state heading | No assets match this state |
| Empty state body | Adjust the lifecycle filter or check Runtime/Doctor for setup issues before running actions. |
| Error state | Rindle Admin could not load this component state. Review the missing fixture or token source, then rerun the gallery check. |
| Destructive confirmation | Owner erasure: Type the owner identifier to confirm after reviewing affected assets. |

Copywriting contract:

- Use `Rindle Admin` as the service identity in component examples, navigation labels, empty states, and error states.
- Component examples must use direct operator verbs: `Review`, `Filter`, `Inspect`, `Preview`, `Confirm`, `Repair`, `Erase`.
- Empty states must name the missing data and the next useful action.
- Error states must name the failed source artifact, fixture, or operation and tell the operator what to do next.
- Destructive examples must show collateral preview copy and typed-confirmation copy even though Phase 90 implements the real action flows.

Source: `guides/admin_console_ia.md`; `guides/ui_principles.md`; `REQUIREMENTS.md` ADMIN-04; `88-CONTEXT.md` D-88-12 through D-88-14.

## Interaction Contract

Phase 88 implements the component-level interaction rules that Phase 89 and Phase 90 will consume.

Visual hierarchy contract:

- The primary focal point is the operational task, not a decorative dashboard metric.
- The shell must expose exactly the six locked top-level surfaces: `Home/Status`, `Assets`, `Upload Sessions`, `Variants/Jobs`, `Runtime/Doctor`, and `Actions`.
- Tables, status chips, drawers, dialogs, toasts, empty states, and skeletons must be dense enough for repeated operator use while preserving `44px` targets and clear focus states.
- Component examples must include realistic ready, processing, warning, danger, quarantine, info, empty, loading, and focus states.

| Area | Contract |
|------|----------|
| Navigation | Use task-first IA and service identity: `Rindle Admin`; no analytics-dashboard sprawl, no decorative hero layout. |
| Theme | Theme picker is a first-class component that writes `data-theme="light\|dark\|auto"`; `auto` follows `prefers-color-scheme`. |
| Buttons | Primary, secondary, quiet, and destructive variants must show hover, active, disabled, loading, and focus-visible states. |
| Tables | Rows must support state chips, row actions, keyboard focus, compact scanning, and stable dimensions without layout shift. |
| Status chips | Include visible label plus icon or equivalent non-color mark; use token-gated foreground/background pairs. |
| Confirm dialog | Show affected owner/assets before enabling destructive action; typed confirmation remains disabled until the expected value matches. |
| Drawer | Opens from the invoking control/row, preserves orientation, and remains keyboard reachable with visible focus return. |
| Toasts | Use for completed work or recoverable failure; include readable text, visible status mark, and dismiss control. |
| Empty states | Explain missing data and next useful action; avoid decorative illustrations unless generated as self-contained assets and useful to the task. |
| Skeletons | Show only for real pending work; stable dimensions must prevent content shift when loaded. |
| Motion | Use token durations only: press `120ms`, popover `160ms`, toast `200ms`, transition `300ms`; respect `prefers-reduced-motion`; no decorative animation. |
| Accessibility | All controls keyboard reachable; focus-visible uses brand focus tokens; no UI text depends on hover-only disclosure; contrast gates reuse `brandbook/src/contrast.mjs` patterns. |
| Gallery | Gallery must use stable selectors and cover light, dark, and system/auto theme behavior for maintainer screenshot review. |

Source: `88-CONTEXT.md` D-88-05 through D-88-19; `guides/admin_console_motion.md`; `guides/admin_console_ia.md`; `guides/ui_principles.md`.

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none | not applicable; no `components.json`, no React/Next/Vite stack, and Phase 88 forbids shadcn/Radix/Tailwind UI dependencies |
| third-party | none | not applicable; no third-party registries declared and Phase 88 forbids registry components |

Registry contract:

- Do not add shadcn, Radix, Tailwind UI, daisyUI, or third-party UI registries to the shipped console.
- Do not add a runtime icon package; inline/self-contained marks are acceptable only when paired with visible text.
- If a later phase proposes any new runtime UI dependency or third-party registry, stop and escalate because dependency footprint is a recorded high-blast-radius boundary.
