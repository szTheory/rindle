## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → LiveView | tab sort/select events cross to the server; values are enum-bounded |
| browser JS → DOM | the Tabs hook mutates tabindex/aria on existing DOM nodes only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-96-03 | Tampering | sort/tab phx-click params | mitigate | Server handlers (Plan 03) validate `sort_key`/tab id against the known column/tab set; no dynamic eval. Function components use `values:` enums. |
| T-96-04 | Denial of Service | Tabs keyboard hook | accept | Hook only reorders focus/tabindex on present nodes; no network, no unbounded loop. Static reference page. |

No high-severity threats: hand-authored CSS + Phoenix function components + a keyboard-only
hook, no data mutation, no auth surface.
</threat_model>
