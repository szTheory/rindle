## Color

Cohort's `--ck-*` palette is **authoritative** and stays separate from `rindle-admin`. The
existing 60/30/10 split is restated; the **dark values move from a `prefers-color-scheme` media
query to a `[data-theme="dark"]` attribute contract** (see Theme Contract below), with
`prefers-color-scheme` retained only as the auto/default fallback.

| Role | Light value | Dark value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--ck-bg` `#f7f8f6` (+ emerald radial glow) | `--ck-bg` `#0b1411` | App background / page shell |
| Secondary (30%) | `--ck-surface` `#ffffff`, `--ck-surface-2` `#fbfcfb`, `--ck-tint` `#ecfdf5` | `--ck-surface` `#111d18`, `--ck-surface-2` `#0f1a15` | Cards, panels, nav, stat tiles, toolbars, table rows |
| Accent (10%) | `--ck-brand` `#059669` / `--ck-brand-strong` `#047857` / `--ck-accent` `#10b981` | `--ck-brand` `#10b981` / `--ck-brand-strong` `#34d399` | See reserved-for list |
| Destructive / quarantine | `--ck-quarantine` `#b91c1c` | `--ck-quarantine` `#f87171` | Quarantine/failed status only (status surface, not an action this phase) |

**Accent reserved for:** primary pill button (`.ck-btn--primary`), the active nav item underline
(`aria-current="page"`), `:focus-visible` rings (`--ck-focus`), card hover border, eyebrow/section
brand marks, and stat-tile accent rules. **Never** applied to all interactive elements â€” quiet
buttons, table headers, and form inputs stay neutral surface/border.

Status colors (`--ck-ready` / `--ck-processing` / `--ck-quarantine` / `--ck-info`) are **always
paired with an icon or text label** (existing `.ck-badge` rule), never color-alone.

### Theme Contract (net-new â€” `[data-theme]`)

Today `cohort.css` only has `@media (prefers-color-scheme: dark)` and **no** `[data-theme]`
selector. This phase authors an explicit, switchable contract on the page root:

| Selector | Resolves to |
|----------|-------------|
| `:root` / `[data-theme="light"]` | Light token set (default) |
| `[data-theme="dark"]` | Dark token set (the values currently in the media query) |
| `@media (prefers-color-scheme: dark)` (when no explicit `[data-theme]`) | Dark token set (auto fallback only) |

Dark is **semantic elevation, not color-inversion** (anti-feature guard): dark surfaces step
LIGHTER with elevation (`--ck-bg` â†’ `--ck-surface-2` â†’ `--ck-surface` â†’ overlay), shadows barely
read on dark so separation comes from tint, never from re-using light drop-shadows. Mirrors the
Phase-94 admin elevation philosophy (PIPE-02) in Cohort's own vocabulary â€” coherent, not shared.

### Color-literal removal (grep-clean target â€” COHORT-06 / Success Criterion 3)

All color **at usage sites** must reference `--ck-*` tokens; the `:root` / `[data-theme]` blocks
are the single literal sink. Current offenders to eliminate this phase:

| Location | Current literal | Replace with |
|----------|-----------------|--------------|
| `.ck-copy[data-copied="true"] { color: #fff }` | `#fff` | `--ck-on-brand` |
| `--ck-bg-glow`, `--ck-shadow-sm/md/lg` rgba literals | raw `rgba(...)` baked per-theme | derive from a `--ck-shadow-ink` / `--ck-glow-ink` base token per theme so shadow/glow tokens compose, not hard-code |
| Any new primitive (table/stat/form/tabs/detail/toolbar) | â€” | must ship token-only; zero new hex/rgb at usage sites |

Grep gate (the mechanical check): a hex/`rgb(`/`rgba(` scan over `cohort.css` finds color literals
**only** inside the `:root` and `[data-theme]` token-definition blocks â€” never in a rule body.

## Motion Contract (net-new `prefers-reduced-motion: reduce`)

Today `cohort.css` has only a `prefers-reduced-motion: no-preference` block (the positive, wrapping
the `.ck-reveal` rise). It has **no `reduce` block** â€” that is net-new this phase (COHORT-06).

| Rule | Contract |
|------|----------|
| Default motion | `transform`/`opacity` only (GPU); sub-300ms; no `transition: all`; no bounce. Existing transitions (`.ck-btn`, `.ck-card`, nav links, copy button, `.ck-reveal` 0.55s rise) conform. |
| `@media (prefers-reduced-motion: reduce)` (NEW) | `*` inside `.ck`: `animation: none !important; transition: none !important;` and `.ck-reveal { opacity: 1; transform: none; }` so reveal content is immediately visible. Hover transforms (`translateY`) collapse to none; color/border state changes are kept (they convey state, not decoration). |
| New primitives | Tabs/sortable-header/toolbar state changes use `--ck-*` motion durations â‰¤ 300ms, and every animated property is covered by the `reduce` block. |

Motion philosophy: materialization, not entertainment (anti-feature guard: no animate-everything).
The `/styleguide` gallery renders one panel under simulated `reduce` to prove the contract.

## Machine-Verifiable Acceptance (for gsd-ui-checker & gsd-ui-auditor)

The contract above is mechanically verifiable; this phase's gates:

1. **Component existence:** all six Level-1 primitives (`.ck-table`, `.ck-stat`, form set,
   `.ck-tabs`, `.ck-detail`, `.ck-toolbar`) + the four Level-2 compositions exist in `cohort.css`
   **and** `CohortComponents`, and are reachable at `/styleguide`.
2. **Theme contract:** `[data-theme="dark"]` selector exists in `cohort.css` (net-new), is

   zeroes animation/transition on `.ck` descendants; `.ck-reveal` resolves to its final state.
4. **Grep-clean tokens:** a hex/`rgb`/`rgba` scan of `cohort.css` finds color literals **only**
   inside `:root` / `[data-theme]` token-definition blocks (no rule-body literals; `#fff` at the
   copy button is replaced by `--ck-on-brand`).
5. **Contrast:** new light + dark Cohort `contrast_pairs` are added to `tokens`/`contrast.mjs` and
   pass at their stated minimums in both themes.
6. **Polish harness:** the page shell carries `data-ck-root`; `admin-polish.js` run with
   `{ root: "[data-ck-root]", interactiveSelectors: [".ck-btn", ".ck-tab", ".ck-input", ...] }`
   reports zero contrast/target/clip offenders over the styleguide (D-94-07 seam; flipped
   warnâ†’fail as the single gate in Phase 102).
