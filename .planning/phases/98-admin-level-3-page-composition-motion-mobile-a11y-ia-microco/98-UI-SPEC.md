> mountable Phoenix LiveView admin surfaces (Overview · Assets · Upload sessions ·
> Processing · Doctor · Maintenance). Generated by gsd-ui-researcher, verified by

> (`test/brandbook/admin_design_system_validation_test.exs`).

## A. Page Composition & Visual Hierarchy — UPLIFT-03

**Affected files:** `lib/rindle/admin/components.ex` (new `page/1` scaffold),
`priv/static/rindle_admin/rindle-admin.css`, all six `*_live.ex` surface modules.

Introduce ONE Level-3 page scaffold `page/1` with named slots. This single grammar is what
makes all six surfaces one system and **structurally forbids page-local styling** (no per-page
wrapper divs carrying grid/measure).

**Canonical vertical order** (top → bottom):
page header (from existing `shell`) → `:summary` triage band → `:filters` → `:work`
(primary table, **required**) → `:aside` inspector → `:actions` footer.

**Slot / attr API:**

```elixir
slot :summary
slot :filters
slot :work, required: true
slot :aside
slot :actions
attr :state, :atom, default: :ok   # :ok | :empty | :error | :loading
```

`:state` makes the scaffold render the matching L1/L2 empty / error / loading fallback so
pages NEVER re-implement them.

**Grid / measure:**
- `:work` single-column by default.
- `:work + :aside` two-pane on lg+ via `grid-template-columns: minmax(0,1fr) minmax(320px,380px)`,
  collapsing to one column below 1024px.
- Text regions capped ~68–78ch; tables full-width.
- Page padding `--rindle-space-6` (32); section gap = `--rindle-space-fluid-section`;
  card gap `--rindle-space-5` (24); intra-card `--rindle-space-4`/`-3`.

**Hierarchy (token-only, assertable):**
1. 1st-order `:summary` → `--rindle-surface-raised` + `--rindle-radius-panel` (20) +
   `--rindle-shadow-card`; h2 display family weight 600.
2. 2nd-order `:work` table → base `--rindle-surface` + `--rindle-radius-card` (14).
3. 3rd-order → secondary text + mono IDs.
- 60/30/10: surface/surface-raised dominate, deep-current structural, accent ≤10% and never
  text on light.

**Density / detail composition:**
- Table row rhythm `--rindle-space-2` block / `--rindle-space-4` inline cell padding.
- Detail composes as the `:aside` two-pane inspector on lg+, degrading to a **dedicated detail
  route** below md.
- AVOID drawers/modals for detail and inline row-expand (both violate the motion/phx-patch
  constraints).

**AVOID:** per-page wrapper divs carrying grid/measure (that IS banned page-local styling —
push into scaffold); Grafana-style free-form panel grids; >1 content measure per surface.

> **Merge-gate assertion (Composition):** the scaffold root exposes the canonical slot order
> in DOM order; `:work` is required (compile error if absent). Computed `grid-template-columns`
> on the two-pane region resolves to two tracks at ≥1024px and one track below. `:summary`
> computed `background` = `--rindle-surface-raised`, `border-radius` = 20px, `box-shadow` =
> card token; `:work` table `border-radius` = 14px. No surface module declares a grid/measure
> wrapper (no page-local `display:grid` outside the scaffold selector).

## B. Motion Contract — UPLIFT-04

**Affected files:** `priv/static/rindle_admin/rindle-admin.css` (animated selectors +
reduced-motion block), `lib/rindle/admin/components.ex` (`JS.transition` wiring), surface
modules using streams/overlays.

**Mechanism:** CSS-state-driven + `Phoenix.LiveView.JS.transition` 3-tuple
`{active, start, end}` with `:time` EQUAL to the CSS duration token. Enter via `phx-mounted`,
leave via `phx-remove` (LiveView holds the node `:time` ms). Menus/popovers via `JS.toggle`
(node stays mounted, no round-trip). Stream-row leave: drive `JS.hide` with transition BEFORE
`stream_delete`.

**Closed motion catalog** (tokens from `tokens.json` `motion`; easings: standard
`cubic-bezier(0.2,0,0,1)`, decelerate `cubic-bezier(0,0,0,1)`, accelerate
`cubic-bezier(0.3,0,1,1)`):

| Element | Moves? | Duration | Easing | Property |
|---------|--------|----------|--------|----------|
| Button / control press | YES | press 120ms | standard | `transform: translateY(1px)` |
| Popover / menu / theme-picker open-close | YES | popover 160ms | decelerate in / accelerate out | opacity + transform |
| Toast / alert enter-exit | YES | toast 200ms | decelerate / accelerate | opacity + transform translateY |
| Drawer / dialog / inspector open-close | YES | transition 300ms | decelerate / accelerate | opacity + transform translateX/Y |
| Stream row insert / remove | YES (restrained) | toast 200ms | decelerate / accelerate | **opacity-only** (no slide) |
| Status-chip state change | MINIMAL | press 120ms | standard | opacity cross-fade of CONTENTS only — NO color tween, NO pulse |
| Skeleton → content swap | YES | transition 300ms | standard | opacity cross-fade (skeleton is STATIC, no shimmer loop) |
| Focus-ring | NO | — | — | instant, never animates |
| Page / surface transition | NO | — | — | instant |
| `live_indicator` | NO | — | — | static, no breathing/pulse loop |

**GPU-only:** ONLY `transform` / `opacity` ever animate — never width/height/top/left/margin
or color/background-color. Elevation via static `box-shadow` token swap (instant). NO
indeterminate spinners (they fake progress — violates "real state, not fake progress"); show
real lifecycle state labels instead.

**phx-patch footgun:** NEVER `transition: all` on any node inside a `phx-update="stream"` or
`phx-patch` region; enumerate exact properties, e.g. `transition: opacity 200ms, transform
200ms`. Audit `rindle-admin.css` for `all` (current audit: no `transition: all` present —
keep it that way).

**Reduced-motion:** single rule —
`@media (prefers-reduced-motion: reduce)` collapses ALL animated selectors to
`transition-duration: 0ms; animation: none; transform: none`. Collapse to instant, keep no
fade.

> **Merge-gate assertion (Motion):** for every animated selector, computed `transition-property`
> ⊆ {`opacity`, `transform`} (assert NO `all`, no width/height/top/left/margin/color/
> background-color). Under emulated `prefers-reduced-motion: reduce`,
> `getComputedStyle(...).transitionDuration === "0s"`; under no-preference, it equals the token
> duration (e.g. `0.2s` for toast). No selector inside a stream/patch region declares
> `transition: all`.

## C. Responsive Contract — UPLIFT-05

**Affected files:** `priv/static/rindle_admin/rindle-admin.css` (convert desktop-first rule at
L1087 to mobile-first; layer min-width queries at the four locked stops), `components.ex`
(nav disclosure button), surface tables.

**Authoring rule:** mobile-first; layer `min-width` media queries at the four locked stops
(convert the current desktop-first `max-width: 760px` rule at `rindle-admin.css` L1087 to
mobile-first). Media queries are the DEFAULT mechanism — NOT container queries (the shell is
one fixed layout; `container-type` would also break the sticky table header). Breakpoints are
literals in queries (CSS conditions can't read custom props) but mirror `tokens.json`
`breakpoint` sm480/md760/lg1024/xl1280.

| Stop | Layout |
|------|--------|
| base 0–479 | single-column; nav stacked; tables → stacked "label: value" cards; drawer/filters = full-screen sheet |
| `min-width:480` (sm) | filter fields 2-up |
| `min-width:760` (md) | **structural switch**: shell becomes grid sidebar `minmax(220px,260px) minmax(0,1fr)`; tables become real `<table>` rows |
| `min-width:1024` (lg) | drawer side-docks at `min(100%, 520px)` |
| `min-width:1280` (xl) | main honors max-width cap |

**Nav transform:** below md, left sidebar `.rindle-admin-nav` becomes a sticky **top bar** with
a disclosure (☰) button toggling the existing `__list` (button has `aria-expanded` /
`aria-controls`, 44px hit-area, `JS.toggle`). NOT a bottom tab bar (6 surfaces > the 3–5 a
bottom bar tolerates). Keep the current surface label visible.

**Dense-table strategy (the hard call):** gov.uk-style STACKED "label: value" cards below md,
real `<table>` at md+, driven purely by CSS off the existing `scope="col"` headers — NO markup
fork, NO priority-column hiding (operators must NEVER silently lose State/Job columns). Keep
semantic markup; flip `table`/`tr`/`td` to `display:block` at narrow width and surface each
column header via `data-label` + `content: attr(data-label)` so each row reads as a
`metadata_list`-shaped card (one card visual language). **ONE exception:** the sticky
internal-scroll table `.rindle-admin-table--sticky` keeps horizontal scroll for wide numeric
matrices.

**Drawers / dialogs / filters:** below md, inspector drawer = full-screen sheet
(`width:100%; min-height:100vh`), reverting to side-docked `min(100%, 520px)` at lg; filter
form stacks single-column (each control full-width, 44px); confirm dialog
`width: min(100%, calc(100vw - var(--rindle-space-4) * 2))` so it never overflows a 360px phone.

**Touch targets:** `--rindle-admin-target-min: 44px` already on nav items / buttons / inputs /
tabs / theme options; close gaps on the new disclosure button, stacked-table row tap target,
and sort-th control. Fluid gutter/section tokens own all page padding — drop fixed
per-breakpoint padding overrides.

> **Merge-gate assertion (Responsive):** at viewport <760px, computed `display` of the data
> `<table>` / `<tr>` / `<td>` = `block` and each `<td>::before` `content` resolves to its
> `data-label`; at ≥760px they are `table`/`table-row`/`table-cell` and `::before` content is
> empty. Shell `grid-template-columns` is single track <760 and two tracks ≥760. Disclosure
> button computed min-height/min-width ≥ 44px and carries `aria-expanded`. `.rindle-admin-table
> --sticky` retains `overflow-x: auto` at all widths. No state/job column has `display:none` at
> any breakpoint.

## D. Accessibility Contract — UPLIFT-06

**Affected files:** `components.ex` (focus_wrap overlays, live regions, theme_picker,
live_indicator, table, skip-link), surface modules with confirm/destructive flows,
`rindle-admin.css` (`:focus-visible`).

**Overlay focus contract:** use `Phoenix.Component.focus_wrap` (NOT a hand-rolled keydown
trap). On open: render `role="dialog"` + `aria-modal="true"` + `aria-labelledby`→title; run
`JS.focus_first`. `focus_wrap` traps Tab/Shift-Tab. ESC closes via `phx-window-keydown` +
`phx-key="escape"`. On close: `JS.focus` back to the stashed trigger id. Background `main` +
`nav` get **`inert` AND `aria-hidden`** while open (aria-modal alone doesn't stop sighted
keyboard tabbing). Interactive confirms use `role="alertdialog"`.

**ARIA per primitive (decisive):**

| Primitive | Contract |
|-----------|----------|
| `status_chip` | non-focusable `<span>`, always renders visible text → NO role/aria-label (redundant-to-text). NEVER `role="status"` (would spam on stream patch). Add `aria-label` only if a chip ever lacks adjacent text. |
| `theme_picker` | **FIX bug:** `aria-pressed` is client-JS-only today (desyncs on dead-render/reconnect). Server owns the attribute (`aria-pressed={@theme == "light"}` …) OR model as `role="radiogroup"`/`role="radio"`+`aria-checked`. JS may enhance, not be source of truth. |
| `nav` | `aria-current="page"` ✓ keep. Add skip link. |
| `live_indicator` | **FIX bug:** drop `tabindex="0"` (dead tab stop on non-interactive `<p>`); make it the POLITE live region `role="status"` (`aria-live="polite"`, `aria-atomic="true"`). Keep decorative span `aria-hidden`. |
| `table` | **FIX bug:** add `<caption>` (visually-hidden ok) naming the surface; `<thead>` with `<th scope="col">`; `scope="row"` on the row-header cell. |
| `filters` | `fieldset`/`legend`/`label` ✓ keep. |
| `empty_state` | static `section` + `h2` ok. |
| `error_state` | announced via ASSERTIVE alert region when it appears async. |

**Live regions (two persistent, server-rendered, present at mount, initially empty):**
- POLITE `role="status"` (`aria-live="polite"`) = `live_indicator` for routine state flips
  ("Variant ready").
- ASSERTIVE `role="alert"` = run-failure / action-error banner.
- Regions must exist BEFORE content arrives. Do NOT put `aria-live` on the
  `phx-update="stream"` table (every row would announce — spam); announce a single summarized
  message ("3 variants ready", "Run 1f2 failed") into the dedicated region.

**Visible focus, both themes:** `:focus-visible` (keyboard-only, NOT `:focus`) →
`outline: 2px solid var(--rindle-focus-ring); outline-offset: 2px`. Ring resolves per theme
(light deep-current 11.34:1, dark rindle-green 9.38:1). A programmatically-focused dialog
container may not trigger `:focus-visible` → give the dialog a permanent visible
border / `:focus` ring.

**Keyboard:** tab order = DOM order. Add skip-link as the FIRST focusable child of shell →
`href="#rindle-admin-main"`; give `<main>` `id="rindle-admin-main"` + `tabindex="-1"`. No traps
except the intended modal trap. Power-user shortcuts: deliberately NONE for v1 (least surprise;
avoids SR virtual-cursor collisions; WCAG 2.1.4).

**Flagged current bugs to fix this phase:** (1) confirm/destructive flows are inline panels
with no trap/ESC/return-focus; (2) `theme_picker` JS-only `aria-pressed`; (3) `live_indicator`
dead `tabindex` + missing `role="status"`; (4) `table` missing caption/thead/scope; (5) no
skip-link, `<main>` missing id/tabindex.

> **Merge-gate assertion (A11y):** every `contrast_pairs` ratio ≥ min in BOTH themes (text 4.5,
> non-text 3.0). Computed `:focus-visible` outline = `2px solid` + `outline-offset: 2px` with
> ring color matching `--rindle-focus-ring` per `data-theme`; mouse-click focus yields NO ring.
> DOM asserts: skip-link is first focusable child → `#rindle-admin-main`; `<main>` has
> `id="rindle-admin-main"` + `tabindex="-1"`; data `<table>` has `<caption>` + `<th scope>`;
> `live_indicator` has `role="status"` + NO `tabindex`; open dialog sets `inert` + `aria-hidden`
> on `main`+`nav` and has `role="dialog"`/`alertdialog` + `aria-modal` + `aria-labelledby`;
> `theme_picker` `aria-pressed` present in server-rendered (dead) markup.

## E. Information Architecture — gov.uk task-first — UPLIFT-07

**Affected files:** `home_status_live.ex` (triage home), `components.ex` `shell` nav
labels/order + skip-link, all surface modules (deep-link query params), `actions_live.ex`
(distribute → contextual actions / Maintenance bucket).

**Triage-home (Overview) ordered sections — problems FIRST:**
1. **"Needs attention"** band — a GDS task-list of ONLY non-zero problem counts, each a
   deep-link:
   - "12 failed processing runs →" → `variants-jobs?state=failed`
   - "8 stale variants →" → `variants-jobs?class=stale`
   - "3 quarantined assets →" → `assets?state=quarantined`
   - "5 expired upload sessions →" → `upload-sessions?state=expired`
   - "2 orphaned objects →" → `runtime-doctor`
2. **System health:** 3 chips (lifecycle-events-flowing · Doctor pass/fail · storage
   reachable).
3. **Recent lifecycle activity:** last 5 events.
4. Vanity totals go below the fold. NEVER render `inspect/1` to an operator. Data source:
   `Queries.home_status/1` (already returns recommendations + counts).

**Nav labels/order (relabel for task-scent, drop slashes):**
Overview (was Home/Status) · Assets · Upload sessions · Processing (was Variants/Jobs) ·
Doctor (was Runtime/Doctor) · Maintenance (was Actions).

"Actions" as a verb-bucket is a GDS junk-drawer anti-pattern → **DISTRIBUTE it:**
- regenerate → on the stale-variant row (Processing)
- release / quarantine → on asset detail (Assets)
- reconcile → on Doctor

"Maintenance" keeps ONLY genuinely contextless cross-cutting ops (owner-erasure /
batch-erasure, GDPR-driven).

**Progressive disclosure, 3 tiers (expert-calibrated):**
dense scannable INDEX (Oban-Web-style — state chips + filter bar; experts want density not
pagination) → one-thing-per-page DETAIL (the "thing" is the decision: one run's
`error_reason` / attempt / worker + its single primary action) → inline DRAWER for the
confirming/destructive fix (operator never loses place, never dead-ends, returns to filtered
list).

**Cross-surface flows:** every finding row self-carries its "do next" deep-link with context
traveling into the action (regenerate stale variants / release quarantine / clean expired
sessions / purge orphaned objects / verify storage — each entry → steps → confirm → return).
Operator never has to re-find the asset after picking a verb.

**Empty / healthy IA:** every surface has an affirmative healthy state, never blank. Overview
all-clear: "Nothing needs attention. Lifecycle events are flowing and all Doctor checks pass."
Distinguish "healthy (reassure)" from "filtered-to-nothing (adjust filters)".

> **Merge-gate assertion (IA):** nav renders exactly six items in order
> [Overview, Assets, Upload sessions, Processing, Doctor, Maintenance] with the relabeled text
> and correct `href`s; no item literally named "Actions"/"Home/Status"/"Variants/Jobs"/
> "Runtime/Doctor". Overview DOM order = needs-attention band → system-health chips → recent
> activity, with vanity totals last. Each needs-attention entry is an `<a>` whose `href`
> carries the documented query param (e.g. `state=failed`). Overview all-clear renders the
> affirmative copy, not the generic filtered-no-match empty state.

## F. Microcopy Contract — operator/SRE voice — UPLIFT-08

**Affected files:** `components.ex` (empty_state/error_state defaults, live_indicator copy),
all surface modules (headers, primary actions, toasts, confirm copy). Brand-book §14 is the
canonical bank — reuse exact strings, fill gaps in-voice.

**Voice rubric (assertable):**
- **R1** action labels ≤4 words, verb-first imperative.
- **R2** status labels single-word Title Case from the frozen lexicon.
- **R3** error/empty bodies state cause + next action (contain an imperative verb).
- **R4** banned hype words (denylist scan): `magically, magic, seamless(ly), effortless(ly),
  powerful, blazing(ly), simply, just, easy, supercharge, next-gen, AI-powered,
  enterprise-grade, worry-free, never worry`.
- **R5** banned vague standalone labels: `Go, Submit, Process, Fix, Continue, Manage, OK`.
- **R6** correct domain noun: variant / upload session / processing run / recipe / profile /
  object / quarantine.
- **R7** calm destructive copy — no "!" in confirmation/destructive bodies.

**Frozen status lexicon** (chips + filters + empty copy use these EXACT strings):
- **Asset:** Staged · Validating · Analyzing · Promoting · Available · Processing · Degraded ·
  Quarantined · Rejected · Detached · Purging · Deleted
- **Variant:** Planned · Queued · Processing · Ready · Stale · Missing · Failed · Purging · Purged
