<objective>
Make every default-suite CI lane run the ExUnit suite EXACTLY ONCE per matrix cell, eliminating the redundant second full-suite run that today doubles test wall-clock and doubles `:epipe` flake exposure on the PR critical path. CI/mix-config + docs ONLY — zero `lib/` change.

The quality lane's single run uses `mix coveralls.multiple --type local --type json --slowest 20`: `--type local` routes through `ExCoveralls.Local.execute/2` → `ensure_minimum_coverage` → the merge-blocking gate is byte-identical to today's `mix coveralls`; `--type json` writes `cover/excoveralls.json` as a side-output of the SAME execution. The integration and adoption lanes drop their redundant standalone `mix coveralls.json` step entirely (decision 2b — no artifact consumer exists; confirmed in D-07).

Purpose: De-flake foundation for milestone v1.21 — halve the `:epipe` exposure surface and make the red/green signal honest (no more "gate passed but job is red" from a flake landing on the second run).
Output: Edited `ci.yml` (3 lanes), one-line `mix.exs` additive config, RUNNING.md doc parity.
</objective>
