<objective>
Make `Rindle.Config.repo/0` consult a `$callers`-aware process-dictionary override BEFORE the
application env, and expose the two test-only setters that write that override. This is the
research-locked foundation (Option (i), ISO-01 + ISO-02) that the counting-double migration (Plan 02),
the guard rule (Plan 03), and the concurrency proof (Plan 04) all build on.

Purpose: eliminate the only mechanism by which the override could ever be set — a global
`Application.put_env(:rindle, :repo, …)` — by giving the resolver a process-local seam (like
Ecto.Sandbox / Mox `$callers`). Adopter-invisible: the default branch is byte-unchanged; an adopter
never sets the override.
Output: `lib/rindle/config.ex` with `repo/0` resolving a per-process override + `put_repo_override/1`
/ `delete_repo_override/0` + private `$callers`-walk helpers.

**Release coupling (D-13 / D-v1.21-01):** this is the phase's only `lib/` touch and must land as a
single `fix:` conventional commit so release-please cuts a **patch** bump to Hex **0.3.2** on green
main. Do NOT use `feat:` (would mis-bump to a minor) or `chore:`/`docs:` (would skip the publish).
</objective>

    - After `Config.put_repo_override(SomeMod)` in the current process: `Config.repo()` returns `SomeMod` in that process.
    - In a `Task.async`/`Task.await` child spawned from a process that set the override: the child's `Config.repo()` returns the parent's override (the child carries `:"$callers"`).
    - In a sibling/unrelated process (NOT a `$callers` descendant, no override of its own): `Config.repo()` returns the app-env value (`Rindle.Repo`), unaffected by another process's override.
    - The `$callers` walk must be cycle-safe (a caller equal to the inspected pid is skipped) and tolerate a dead/unknown pid (`Process.info(pid, :dictionary)` returning non-`{:dictionary, _}` yields `nil`).
  </behavior>
  <action>

    Rewrite `repo/0` so it first consults `repo_override(self())` and only falls through to the existing
    `Application.get_env(:rindle, :repo, Rindle.Repo)` when that is nil (use the `with nil <- repo_override(self()) do … end`
    shape from the sketch so the app-env fallback line is preserved byte-for-byte per D-01).
    Add three private helpers exactly as sketched: `repo_override/1` (consult this pid's dict key, else
    `caller_repo_override/1`); `caller_repo_override/1` (read `:"$callers"`, `List.wrap`, `Enum.find_value`
    over callers that are not the current pid, recursing through `repo_override/1` — this is the cycle guard);
    and `process_get/2` (when `pid == self()` use `Process.get(key)`, else `Process.info(pid, :dictionary)`
    and `Keyword.get`, returning nil on any non-`{:dictionary, _}` result). Keep `@spec repo() :: module()`.
    Do NOT alter any other function in the module. The override-walk cost only runs when an override is
    present; the production hot path is a single `Process.get` returning nil then the unchanged app-env read (D-01).
