<objective>
Add a `:global_repo_swap` rule to the v1.20 async-safety guard
(`test/async_safety_guard_test.exs`) that flags `Application.put_env`/`delete_env(:rindle, :repo, …)`
in ANY test module regardless of async flag — making the global-repo-swap footgun un-reintroducible — and
allowlist the 9 pre-existing legitimate swappers (they swap to an adopter/probe repo to test resolution,
not the counting-double cross-pollution). The structural change (D-08): today the guard only parses
`async: true` module bodies, so the new rule requires a SEPARATE all-modules scan path that applies only
the `:global_repo_swap` classifier and honors the existing `@async_safety_allow` allowlist on those
now-scanned `async: false` modules. ISO-04.

Purpose: a future contributor who reaches for `Application.put_env(:rindle, :repo, …)` gets a red,
merge-blocking gate pointing them at `Config.put_repo_override/1` (research §iv, the enforcement complement
to Plan 01).
Output: the new guard rule + all-modules scan path + 9 allowlisted swapper modules.
</objective>
