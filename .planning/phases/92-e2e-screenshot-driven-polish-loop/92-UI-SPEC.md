# Phase 92 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none - use existing generated `rindle-admin` CSS only |
| Preset | not applicable |
| Component library | none |
| Icon library | none required; status chips use text label plus current-color dot |
| Font | `Space Grotesk` for compact headings, `Atkinson Hyperlegible` for UI/body, `JetBrains Mono` for code/IDs/confirmation strings |

Source: `guides/ui_principles.md`, `brandbook/tokens/tokens.json`, `brandbook/tokens/rindle-admin.css`, and Phase 92 research. Do not add Tailwind, daisyUI, shadcn, Radix, Tailwind UI, or host asset-pipeline dependencies to satisfy this phase.

Phase 92 must test and polish the live Phoenix LiveView console mounted by the Cohort adoption demo at `/admin/rindle`. It must not extend the static gallery as the primary capture target. All Playwright selectors inside the shipped admin package use `data-rindle-admin-*`, not `data-testid`.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Theme-picker inner gaps, chip vertical padding, inline utility gaps |
| sm | 8px | Compact element spacing, nav item gaps, button icon/text gaps |
| md | 16px | Default control horizontal padding, table cell horizontal rhythm |
| lg | 24px | Nav padding, confirm-dialog padding, section clusters |
| xl | 32px | Main shell padding and desktop shell gap |
| 2xl | 48px | Empty-state padding and major page breaks |
| 3xl | 64px | Page-level spacing only; avoid adding new 64px gaps inside dense admin surfaces |

Exceptions: existing `--rindle-space-3: 12px` is allowed for table cell vertical padding and toast gaps because it is already generated from `tokens.json`; status chips may use 28px minimum height; icon/control-only interactive targets must still meet the existing 44px minimum target via `.rindle-admin-target-min`.

Screenshot polish must fail or produce a fix task when any captured screen shows clipped labels, unstable row heights, accidental horizontal scroll at the configured viewport, overlapping controls, or controls below the 44px interactive target.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 17px | 400 | 1.6 |
| Label | 14px | 400 | 1.45 |
| Heading | 22px | 600 | 1.25 |
| Display | 30px | 600 | 1.15 |

Typography contract: use exactly these four sizes in new or adjusted admin UI for this phase. Do not introduce hero-scale 44px/64px text into the operational console. Confirmation strings such as `ERASE owner_type:owner_id` and `ERASE N OWNERS` render in the mono family and must remain readable in both light and dark screenshots.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--rindle-surface`: light `#F7F4EA`, dark `#0E1316` | Page background and main shell field |
| Secondary (30%) | `--rindle-surface-raised`: light `#FBFEFC`, dark `#161E23` | Nav, tables, empty/error states, toasts, dialogs, action panels |
| Accent (10%) | `--rindle-brand`: light `#123A35`, dark `#32D08C` | Primary button fill, active theme option, current/focus brand affordances |
| Destructive | `--rindle-status-danger`: light `#C83232`, dark `#F09090` | Destructive actions only: owner erasure, batch erasure, destructive error/warning states |

Accent reserved for: primary "Review assets" CTA, active theme option, focus ring via `--rindle-focus-ring`, selected/current navigation affordance, and dark-theme brand text. Do not use `#32D08C` for normal text on light surfaces; token rules mark it accent/large-graphic-only there.

Status colors must always be paired with a visible label or current-color dot. Screenshots must include ready, processing, warning, danger, quarantine, info, empty, and error states when seed data or route state exposes them.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Review assets |
| Empty state heading | No records match this view |
| Empty state body | Adjust the filters or review Runtime/Doctor to confirm Rindle is receiving lifecycle events. |
| Error state | Rindle Admin could not load this surface. Review the runtime checks, then retry after the missing source is available. |
| Destructive confirmation | Owner erasure: Type `ERASE {owner_type}:{owner_id}` to confirm. Batch erasure: Type `ERASE {count} OWNERS` to confirm. |

Phase 92 Playwright coverage must assert destructive copy and interaction states, not only final receipts:

- Owner erasure preview appears before execution and shows affected collateral.
- Batch erasure preview appears before execution and shows owner count plus affected collateral.
- Wrong confirmation keeps the user in preview/input state and surfaces `Confirmation does not match.`
- Receipts state what changed and what did not change.
- Quarantine review remains read-only triage copy; do not introduce an un-quarantine mutation.

Screenshot filenames and labels should use the surface slug, not decorative prose: `home-status`, `assets`, `asset-detail`, `upload-sessions`, `upload-session-detail`, `variants-jobs`, `runtime-doctor`, `actions`, `actions-owner-preview`.

## Interaction Contract

| Surface / State | Required Contract |
|-----------------|-------------------|
| Shell | Every route shows `[data-rindle-admin-root]`, `[data-rindle-admin-surface]`, nav, page header, live indicator, and theme picker. |
| Theme picker | Playwright toggles `[data-rindle-admin-theme="light"]`, `[data-rindle-admin-theme="dark"]`, and `[data-rindle-admin-theme="auto"]`; selected option sets `aria-pressed="true"` and the root `data-theme`. |
| Navigation | Every nav item remains at least 44px tall, exposes the current surface with `aria-current="page"`, and uses task names already shipped in `SURFACES`. |
| Tables | Rows use stable `data-rindle-admin-row` selectors; row hover/focus cannot resize rows or hide chips. |
| Detail pages | First seeded asset and upload-session detail pages must render redacted provider/session values through `data-rindle-admin-redacted-value`; no raw secret/provider URI is visible. |
| Empty and error states | Empty state uses `data-rindle-admin-empty-state`; error state uses `data-rindle-admin-error-state`; both are captured in light and dark when reachable. |
| Destructive flows | Preview before execution, exact typed confirmation, blocked wrong confirmation, destructive styling, and receipt are all required. |
| Motion | Only existing tokenized press/popover/toast/transition motion is allowed; tests should disable animations for screenshots and never add sleeps. |

Capture matrix: full-page screenshots at 1480x900 for every listed surface in light and dark. Also capture a mobile/narrow viewport for the shell/nav and Actions surface if the plan can keep CI runtime within the existing adoption-demo E2E lane. Output stays under ignored Playwright/test-results paths such as `examples/adoption_demo/test-results/admin-screenshots/{theme}/{surface}.png`; CI must assert every expected file exists.
