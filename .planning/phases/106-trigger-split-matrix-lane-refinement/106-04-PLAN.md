<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cron tick → release pipeline | A scheduled run named `CI` on head_branch main would trip release-please-automerge → unattended release. The separate `Nightly` file must stay invisible to it |
| scheduled run → github issues | nightly-failure-issue writes issues; the `issues: write` token must be scoped to that job only |
| nightly job → secrets (GCS) | gcs-soak / package-consumer-gcs-live use repo secrets; gated `if: github.repository == 'szTheory/rindle'` so forks never run them |
| PR event → nightly lane | nightly.yml has no `pull_request:` trigger, so it can never become a PR required check |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-106-04-SR | Spoofing / Repudiation (unattended release) | a `schedule:` on a workflow named `CI` on head_branch main firing release-please-automerge's `workflow_run:[CI]` listener on a cron tick → auto-merge + dispatch release.yml | mitigate (HIGH) | D-12: the nightly lane is a SEPARATE file `nightly.yml` with run name `Nightly` and distinct workflow_id, NO `schedule:` on ci.yml, NO `pull_request:`; release-please-automerge's `workflows:[CI]` filter never matches `Nightly`, so a clock tick can never trigger a release |
| T-106-04-E | Elevation of privilege | nightly-failure-issue token scope | mitigate | Job-scoped `permissions: issues: write` only (no contents:write, no pull-requests:write); top-level workflow default stays `contents: read`; the issue job cannot push code or merge PRs |
| T-106-04-T | Tampering (script injection) | inline `gh issue` body interpolating untrusted text on a `schedule` event | mitigate | Schedule events carry no attacker-controlled PR title/body; issue body is composed from static text + `needs.*.result` (trusted context), not interpolated user input into a shell; no `run:` step echoes untrusted event fields |
| T-106-04-I | Information disclosure | GCS secrets in moved jobs | accept | Same secret-gated, repo-restricted posture as before the move; secrets unchanged; fork PRs never reach these jobs (now nightly-only) |
| T-106-04-FG | Tampering ("fake green") | Dialyzer/gcs-live continue-on-error masking nightly red | mitigate | Dialyzer is gating in nightly (no continue-on-error, D-17); package-consumer-gcs-live's continue-on-error is DROPPED on move (D-14); failures surface via Nightly Summary + nightly-failure-issue (D-16). No continue-on-error is added to any ci.yml lane feeding the push:main run conclusion |
| T-106-04-SC | Tampering (supply chain) | nightly compat-matrix `mix deps.get` across older OTP/Elixir | accept | Deps resolved from committed mix.lock; no new package introduced. Action SHA-pinning of nightly.yml's actions is deferred to Phase 107 (HARD-02) — noted, not implemented |
