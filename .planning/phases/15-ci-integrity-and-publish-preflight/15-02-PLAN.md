| Boundary | Description |
|----------|-------------|
| local maintainer shell → GitHub Actions CI | Local diagnostics inform iteration, but authoritative release proof must come from remote CI on the exact commit |
| repository automation → Hex account/package state | Repo code cannot prove maintainer identity, owner authority, or package-name availability without human confirmation |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-15-05 | Repudiation | release-candidate signoff evidence | mitigate | Record exact SHA, CI run URL, and checklist completion in a phase-local artifact and summary so Phase 16 can trace the go/no-go source. |
| T-15-06 | Tampering | release proof source | mitigate | Treat only the GitHub Actions run for the exact release-candidate SHA as authoritative; local runs are documented as supplementary diagnostics only. |
</threat_model>
