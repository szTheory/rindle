<objective>
Land the Option-b1 `:epipe` absorption shim at the single AV chokepoint and prove it with a
merge-blocking regression suite. `Rindle.AV.Subprocess.run/3` delegates to a new `@doc false def
run_isolated/5` that owns the MuonTrap port inside a throwaway `spawn_monitor`'d, `trap_exit`'d
worker, so a MuonTrap #98 broken-pipe (`{:EXIT, port, :epipe}`) async exit dies with the worker
and can never kill the caller — while the `{output, status}` contract and security invariants 8–13
stay byte-equivalent by construction.

Purpose: every AV invocation (tests AND adopter Oban workers) becomes deterministic; ships
adopter-invisible `fix:` → Hex 0.3.2 (D-v1.21-01). The approach is research-LOCKED to Option b1
(D-01..D-07); this plan TRANSLATES the locked design — it does not redesign it.
Output: the shim + `require Logger` + NOTE block in `subprocess.ex`; the `:canary` exclude in
`test_helper.exs` (load-bearing safety for Plan 02); the merge-blocking regression test file.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BEAM process ↔ MuonTrap port (OS subprocess) | The `:epipe` async exit crosses here when MuonTrap's ACK write hits a closed reader after the child exits (MuonTrap #98). |
| caller (Oban worker / test) ↔ `run/3` | The boundary the shim protects: the async transport signal must NOT cross from the port into the caller. |
| argv construction ↔ subprocess invocation | BYTE-FROZEN this phase: `build_args/3`/`build_opts/2`, no-shell argv-list, MuonTrap parent-death kill + Linux cgroup caps (invariants 8–13). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-109-01 | Denial of Service | `Subprocess.run/3` ← MuonTrap port `:epipe` (the bug) | high | mitigate | b1 shim: port owned by a throwaway `trap_exit`'d worker; the `:epipe` `{:EXIT, port, _}` dies with the worker; parent monitors (never traps); real `{output, status}` preserved. Proven by the merge-blocking regression suite (Task 3). |
| T-109-02 | Tampering | argv construction (`build_args/3`/`build_opts/2`) | low | accept | BYTE-FROZEN this phase (invariant 8 allowlist + `-protocol_whitelist file,crypto,data`, no shell). Security surface unchanged by construction; the shim sits BELOW run/3 and ABOVE MuonTrap.cmd, never on the argv path. |
| T-109-03 | Denial of Service | orphaned child after BEAM death | low | accept | Unchanged — MuonTrap stdin-EOF C kill (invariant 13) fires on BEAM (OS-process) death, not BEAM-process death; running MuonTrap.cmd from the worker is irrelevant to the kill guarantee (RESEARCH §7.1). |
| T-109-04 | Tampering | npm/pip/cargo/hex installs | n/a | accept | No new dependency added (D-03: no Rambo/exile/Task.Supervisor; MuonTrap pin unchanged). No package-legitimacy gate needed. |
