Ship the generated-app `:mux` profile mode harness extension (MUX-18) — the
heavy lane of Phase 36. This plan extends `test/install_smoke/support/generated_app_helper.ex`
with a third profile discriminator (`:mux` alongside `:image | :video`),
adds a new `Rindle.InstallSmoke.GeneratedAppSmokeMuxTest` test module mirroring
the existing Image/Video modules, extends `scripts/install_smoke.sh` to accept
`mux`, wires a new `mux-enabled` cassette step into the existing
`package-consumer` GitHub Actions job, adds a sibling `mux-soak` top-level job
behind a `streaming` PR label gate, generates and commits the
`test_signing_public_key.pem` fixture, and ships the `scripts/mux_soak_cleanup.sh`
belt-and-suspenders cleanup script.

This plan has a HARD dependency on Plan 01 — the cassette lane uses
`Rindle.Profile.Presets.MuxWeb` as the generated app's video-style profile.
The doc-parity guard's required-strings list (`Rindle.Profile.Presets.MuxWeb`)
is owned by Plan 02; this plan does NOT re-edit that list. This plan touches
`.github/workflows/ci.yml` for FOUR distinct edits — none of which overlap with
Plan 02's doc-parity-guard edit (which is at lines 518-545; this plan edits
lines 3-7 trigger types, the `package-consumer` step block at line 376-377,
and appends a new `mux-soak` job).

Purpose: Adopters get a reproducible "Mux works in a fresh Phoenix app"
proof on every PR (cassette mode, free, fast), AND maintainers get a
reproducible "real Mux integration is healthy" proof on demand (label-gated
soak mode, free at the v1.6 traffic shape, fork-PR-safe). The split keeps
per-PR cost at zero AND keeps the maintainer-on-demand verification cheap.

Output: Three new files (`scripts/mux_soak_cleanup.sh`,
`test/fixtures/mux/test_signing_public_key.pem`, plus the new
`Rindle.InstallSmoke.GeneratedAppSmokeMuxTest` ExUnit module appended to
the existing test file). Five edits to existing files (`generated_app_helper.ex`
across five sites, `generated_app_smoke_test.exs` test-module append +
`assert_install_source!/1` extension, `scripts/install_smoke.sh` one-line
case extension, `.github/workflows/ci.yml` four edits).
