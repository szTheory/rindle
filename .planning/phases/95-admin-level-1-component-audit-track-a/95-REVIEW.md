## Critical Issues

### CR-01: Mobile stacked-table CSS applies to sticky tables and emits blank labels

**Classification:** BLOCKER
**File:** `brandbook/src/admin-css-build.mjs:1178`
**Issue:** The mobile stacked-table rules exclude only a table element with `.rindle-admin-table--sticky`, but the gallery marks the sticky scroll region on the wrapper (`<div class="rindle-admin-table--sticky">`) and leaves the table as only `<table class="rindle-admin-table">` at `brandbook/src/admin-gallery.mjs:196-197`. As a result, the generated CSS still converts sticky data tables to block/card layout below 760px, despite the comment saying sticky tables keep horizontal scroll. The same rules render `td::before { content: attr(data-label); }`, but both the normal gallery table rows and the meta data-table rows emit `<td class="rindle-admin-table__cell">` without `data-label` (`brandbook/src/admin-gallery.mjs:151-157`, `brandbook/src/admin-gallery.mjs:209-234`; generated HTML shows the same at `brandbook/admin-gallery/index.html:307-313` and `brandbook/admin-gallery/index.html:546-571`). That gives mobile users unlabeled stacked cells, and the current gallery check does not catch it because `assertStackedCard` is exported but not invoked by `brandbook/src/admin-gallery-check.mjs`.
**Fix:**
```css
/* Keep sticky-scroll tables in table layout when the marker is on the wrapper. */
.rindle-admin-table--sticky .rindle-admin-table,
.rindle-admin-table--sticky .rindle-admin-table tbody,
.rindle-admin-table--sticky .rindle-admin-table tr,
.rindle-admin-table--sticky .rindle-admin-table td {
  display: revert;
}
```

Also add `data-label` to every non-sticky gallery data cell generated by `tableMarkup` and call `assertStackedCard` from the gallery/browser check at a sub-760 viewport so this cannot regress.
