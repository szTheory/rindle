# Plan 02-04: Oban Workers (ProcessVariant, PromoteAsset)

**Goal:** Implement the Oban workers responsible for promoting assets to an available state and generating their variants asynchronously.

## Success Criteria
- [ ] `Rindle.Workers.PromoteAsset` worker implemented.
- [ ] `Rindle.Workers.ProcessVariant` worker implemented.
- [ ] `PromoteAsset` transitions asset to `available` and enqueues `ProcessVariant` jobs for all profile variants.
- [ ] `ProcessVariant` generates and stores variant, transitions to `ready`.
- [ ] Atomic promote check: Workers abort if the asset's attachment has changed since the job was enqueued.
- [ ] Unit/Integration tests for workers using Oban's testing helpers.

## Proposed Changes

### Workers
- Create `lib/rindle/workers/promote_asset.ex`.
- Create `lib/rindle/workers/process_variant.ex`.

### Core Logic
- `PromoteAsset`:
    1. Reload asset.
    2. Transition `analyzing -> promoting -> available`.
    3. For each variant in profile: create `MediaVariant` (state: `planned`) and enqueue `ProcessVariant` worker.
- `ProcessVariant`:
    1. Reload variant and asset.
    2. Transition variant `planned -> queued -> processing`.
    3. Call processor.
    4. Store result.
    5. Transition to `ready`.
