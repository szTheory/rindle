# Plan 02-01: Direct Upload Broker

**Goal:** Implement the Direct Upload Broker (`Rindle.Upload.Broker`) to manage the direct-to-storage upload lifecycle, including session initiation, URL signing, and completion verification.

## Success Criteria
- [ ] `Rindle.Storage` behaviour includes `head/2` callback.
- [ ] `Rindle.Storage.Local` and `Rindle.Storage.S3` implement `head/2`.
- [ ] `Rindle.Upload.Broker.initiate_session/2` creates an `initialized` session.
- [ ] `Rindle.Upload.Broker.sign_url/1` transitions session to `signed` and returns a presigned PUT URL.
- [ ] `Rindle.Upload.Broker.verify_completion/1` confirms object existence via `head/2`, transitions session to `completed`, and creates a `staged` Asset.
- [ ] Unit tests for `Broker` cover happy path and failure modes (object missing, invalid transitions).

## User Review Required
> [!IMPORTANT]
> This plan introduces a dependency on the storage adapter's `head` capability for verification. If a storage provider does not support `HEAD`, verification will fail.

## Proposed Changes

### Storage Layer
- Update `lib/rindle/storage.ex` to add `@callback head(key, opts)`.
- Update `lib/rindle/storage/local.ex` to implement `head/2` (via `File.exists?`).
- Update `lib/rindle/storage/s3.ex` to implement `head/2` (via `ExAws.S3.head_object`).

### Upload Broker
- Create `lib/rindle/upload/broker.ex`.
- Implement `initiate_session(profile, opts)`.
- Implement `sign_url(session_id)`.
- Implement `verify_completion(session_id)`.

### Schemas
- Ensure `Rindle.Domain.MediaUploadSession` has the necessary fields (matching `media_upload_sessions` table).

## Verification Plan

### Automated Tests
- Create `test/rindle/upload/broker_test.exs`.
- Mock storage `head/2` to simulate presence/absence.
- Assert session and asset state transitions.
- Run `mix test test/rindle/upload/broker_test.exs`.

### Manual Verification
- None required for this plan (unit/integration tests sufficient).
