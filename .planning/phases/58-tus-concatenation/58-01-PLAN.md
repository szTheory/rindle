<task type="auto">
  <name>Task 2: Implement S3 adapter concatenation via UploadPartCopy</name>
  <files>lib/rindle/storage/s3.ex</files>
  <action>
Implement `concatenate(final_key, source_keys, opts)` in `lib/rindle/storage/s3.ex`. Follow the S3 pattern from the research:
1. Initiate a new multipart upload for `final_key` via `S3.initiate_multipart_upload`.
2. Iterate `source_keys` with index (1-based for part numbers). For each source key:
   - Perform a HEAD request `Rindle.Storage.S3.head(src_key, opts)` to get the byte `size`. ExAws `upload_part_copy` strictly requires a Range struct (e.g. `0..(size-1)`).
   - Call `S3.upload_part_copy(bucket, final_key, bucket, src_key, upload_id, part_num, 0..(size - 1))`.
   - Extract the ETag from the response.
3. Complete the multipart upload with the collected parts using `S3.complete_multipart_upload`.
4. Delete all `source_keys` via `S3.delete_object`.
5. Return `{:ok, %{key: final_key, bucket: bucket, upload_id: upload_id}}`.
  </action>
  <verify>
    <automated>mix compile</automated>
  </verify>
  <done>S3 concatenate leverages server-side UploadPartCopy with dynamic sizing.</done>
</task>
