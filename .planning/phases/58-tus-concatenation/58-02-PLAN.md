  <name>Task 2: Implement GCS `concatenate/3` with 32-chunk batch folding</name>
  <files>lib/rindle/storage/gcs.ex</files>
  <action>
Implement `concatenate(final_key, source_keys, opts)` in `lib/rindle/storage/gcs.ex`.
1. The Google Cloud Storage compose API enforces a strict limit of 32 source objects per request. Use `Enum.chunk_every(source_keys, 32)` to batch the keys.
2. If there are multiple chunks (more than 32 total sources), you must fold them: compose the first 32 into a temporary composite object, then use that composite object + the next batch, etc., until you arrive at the `final_key`.
3. For batches <= 32, simply call `Client.compose/4` once.
4. After successfully composing the `final_key`, iterate and delete all the original `source_keys` using `Client.delete/3` (or the equivalent existing deletion function).
5. Return `{:ok, %{key: final_key, bucket: bucket}}`.
  </action>

    <automated>mix compile</automated>
  </verify>
  <done>GCS adapter concatenates objects server-side, respects the 32 object limit, and cleans up sources.</done>
