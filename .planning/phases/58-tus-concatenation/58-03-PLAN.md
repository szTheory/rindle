<task type="auto">
  <name>Task 2: Handle `Upload-Concat: final` session convergence</name>
  <files>lib/rindle/upload/tus_plug.ex, lib/rindle/upload/broker.ex</files>
  <action>
In `handle_post/2`, if the `Upload-Concat` header starts with `"final;"`:
1. Parse the space-separated URLs after `"final;"`.
2. Extract the token from the end of each URL and verify it via `Plug.Crypto.verify` (mimicking `verify_token/2` logic).
3. Delegate to `Rindle.Upload.Broker.concatenate_tus_sessions(tokens, new_upload_opts)` which will:
   a. Load the corresponding `MediaUploadSession`s from the database.
   b. Assert that EVERY session has `%{"is_partial" => true}` in `multipart_parts`, and that each session is fully uploaded (i.e. its `last_known_offset` equals its `upload_length`). Return an error tuple if these assertions fail.
   c. Create a new final `MediaUploadSession`.
   d. Gather the `upload_key` from each partial session (`source_keys`).
   e. Call `Rindle.Storage.concatenate(final_session.upload_key, source_keys, opts)`.
   f. Immediately mark the final session as complete.
4. Handle the return from `Rindle.Upload.Broker.concatenate_tus_sessions`. If successful, return `201 Created` with the final location header. If error, return `400 Bad Request`.
  </action>
  <verify>
    <automated>mix test test/rindle/upload/tus_plug_test.exs</automated>
  </verify>
  <done>POST `final` correctly verifies URLs, asserts partial/complete status, physically concatenates via Storage, and completes the final session.</done>
</task>
