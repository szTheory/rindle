# Phase 94 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase type: FOUNDATION / design-system tokens.** This phase ships no screens. Its contract
> is the **locked token VALUES** for four new categories (motion presets, dark elevation/shadow
> ladder, fluid type/space + named breakpoints, differentiated dark status surfaces) that
> downstream visual phases (95–102) inherit. Every value below is **machine-verifiable** — the
> `brandbook-tokens` job (regen + WCAG contrast + `git diff --exit-code`) and the generalized
> `admin-polish.js` computed-style gate are the merge blockers for this milestone, so this spec
> fixes exact numbers, not ranges. Values target the **`rindle-admin` (BEM, generated)** surface
> only; the parallel `--ck-*` vocabulary is documented as a coherence reference for Phase 96
> (Track B hand-authors Cohort's versions — never a shared build step, per D-94-05/06).

## Spacing Scale

Existing 8-step scale is **unchanged and authoritative** (multiples of 4, source `tokens.json` `spacing`):

| Token | Value | Usage |
|-------|-------|-------|
| `space-1` | 4px | Icon gaps, inline padding |
| `space-2` | 8px | Compact element spacing |
| `space-3` | 12px | Tight stack / control internal |
| `space-4` | 16px | Default element spacing |
| `space-5` | 24px | Section padding |
| `space-6` | 32px | Layout gaps |
| `space-7` | 48px | Major section breaks |
| `space-8` | 64px | Page-level spacing |

**New this phase — fluid SPACE steps (additive, `clamp()` on layout rhythm only).** Two
fluid gutters interpolate between the named breakpoints below; the fixed `space-*` scale is
never replaced (table/control density must stay fixed-px for scannability, per D-94-09):

| Token | clamp(min, preferred, max) | Resolves | Usage |
|-------|----------------------------|----------|-------|
| `space-fluid-gutter` | `clamp(16px, 12px + 1.25vw, 32px)` | 16px → 32px | Page gutter / shell padding across breakpoints |
| `space-fluid-section` | `clamp(32px, 24px + 2.5vw, 64px)` | 32px → 64px | Vertical rhythm between major page sections |

Exceptions: minimum interactive target stays **44px** (`MIN_TARGET_PX`, existing `admin-polish.js` assertion) — not a spacing token, an accessibility floor.

## Named Breakpoints (new this phase)

Locked, named, emitted as tokens so `clamp()` anchors and later media queries reference one
vocabulary. Anchored on the single existing admin breakpoint (760px) so the current
`@media (max-width: 760px)` rule (`admin-css-build.mjs:531`) maps to `--bp-md` without changing output.

| Token | Value | Name | Role |
|-------|-------|------|------|
| `bp-sm` | 480px | mobile | Smallest supported; single-column floor |
| `bp-md` | 760px | tablet | Existing admin collapse point (frozen) |
| `bp-lg` | 1024px | desktop | Sidebar + content two-column |
| `bp-xl` | 1280px | wide | Max content measure reached |

## Color

Existing 60/30/10 split is **unchanged** (source `tokens.json` `color.semantic`). Restated for the checker:

| Role | Light value | Dark value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `warm-shell` `#F7F4EA` | `dark-bg` `#0E1316` | App background / main surface |
| Secondary (30%) | `porcelain` `#FBFEFC` (raised) / `mist` `#E3EAE5` (sunken) | `dark-surface` `#161E23` (raised) | Cards, sidebar, nav, panels |
| Accent (10%) | `deep-current` `#123A35` (brand) · `rindle-green` `#32D08C` / `rind-lime` `#CFEF6A` (graphic accent) | `rindle-green` `#32D08C` | See reserved-for list |
| Destructive | `danger` `#C83232` | `dark-danger` `#F09090` | Destructive actions only |

Accent reserved for: **primary brand buttons, the active nav item, focus rings, and brand
headers/marks.** `rindle-green`/`rind-lime` are accent/large-graphic only on light surfaces
(measured 1.81 / 1.18 contrast — never body text on light; text-capable only on ink/dark). Status
color is always paired with an icon or label, never color-alone (existing rule, kept).

### New this phase — Differentiated dark status SURFACES

Today the six dark status surfaces **collapse** to a single `dark-surface` `#161E23`
(`tokens.json:99–104`), so dark status chips lose their semantic tint. This phase differentiates
them — each is a per-hue tint of the dark base, validated AA against its own dark status TEXT
color and added as new dark contrast pairs to the WCAG gate (D-94-08):

| Status | dark text (existing) | **new** dark surface | Contrast (text on surface) | Min |
|--------|----------------------|----------------------|----------------------------|-----|
| ready | `dark-ready` `#4FD79C` | `dark-ready-surface` `#16241E` | 8.84:1 | 4.5 |
| processing | `dark-processing` `#A99DEB` | `dark-processing-surface` `#1C1E2B` | 6.86:1 | 4.5 |
| warning | `dark-warning` `#E2A85B` | `dark-warning-surface` `#241F16` | 7.79:1 | 4.5 |
| danger | `dark-danger` `#F09090` | `dark-danger-surface` `#2A1A1A` | 7.21:1 | 4.5 |
| quarantine | `dark-quarantine` `#BDAED6` | `dark-quarantine-surface` `#211C2B` | 8.05:1 | 4.5 |
| info | `dark-info` `#85AAF5` | `dark-info-surface` `#161D2B` | 7.28:1 | 4.5 |

All six clear AA with margin (computed via the brand luminance formula; numbers above are the gate's expected values). Light status surfaces are unchanged.

### New this phase — Dark elevation / shadow ladder (semantic, NOT inversion)

A **4-level surface-LIGHTNESS ladder** (FEATURES "dark-mode-done-right": shadows barely read on
dark, so elevation is tint, not heavier shadow — D-94-09). Each step is monotonically lighter than
the last; primary text (`dark-text` `#F1F7F3`) clears AA on every step:

| Token | Hex | Relative luminance | `dark-text` contrast | Role |
|-------|-----|--------------------|----------------------|------|
| `elevation-0` | `#0E1316` (= `dark-bg`) | 0.0062 | 17.21:1 | App base |
| `elevation-1` | `#161E23` (= `dark-surface`) | 0.0122 | 15.54:1 | Raised panel / card |
| `elevation-2` | `#1E282E` | 0.0199 | 13.83:1 | Nested / hover surface |
| `elevation-3` | `#27343B` | 0.0320 | 11.79:1 | Overlay / modal / drawer |

Companion **shadow ladder** (light mode keeps real shadows; dark relies on the tint ladder, with shadows used only for overlay separation):

| Token | Value | Usage |
|-------|-------|-------|
| `shadow-card` | `0 1px 2px rgba(16,20,23,0.06), 0 8px 24px rgba(16,20,23,0.06)` | Existing — cards (light) |
| `shadow-raised` | `0 1px 2px rgba(16,20,23,0.06)` | Subtle raise / sticky header |
| `shadow-overlay` | `0 18px 40px -22px rgba(16,20,23,0.32)` | Drawers, modals, popovers (both themes) |

Anti-pattern guard (out of scope, enforced): no dark-by-color-inversion; no color-only status; elevation is tint, never reused light-mode drop-shadows on dark.

## Motion Presets (new/extended this phase)

Existing four durations + easing are kept and **named as presets**; reduced-motion is a contract,
not a suggestion. `MOTION_TOKENS` (`admin-design-system-data.mjs:44`) must register every key below
or the parity gate omits them silently (D-94-08).

| Token | Value | Reserved for |
|-------|-------|--------------|
| `motion-press` | 120ms | Active/pressed tactile feedback (<150ms) |
| `motion-popover` | 160ms | Menus, tooltips, popovers |
| `motion-toast` | 200ms | Toast enter/exit |
| `motion-transition` | 300ms | Drawer/panel/section transitions (sub-300ms ceiling) |
| `motion-easing-standard` | `cubic-bezier(0.2, 0, 0, 1)` | Default ease (enter/move) |
| `motion-easing-decelerate` | `cubic-bezier(0, 0, 0, 1)` | Elements entering / origin-aware reveals |
| `motion-easing-accelerate` | `cubic-bezier(0.3, 0, 1, 1)` | Elements exiting / reversing |

Rules (contract): materialization not entertainment; `transform`/`opacity` only (GPU); no
`transition: all`; no bouncing; every animated property is wrapped by the existing
`@media (prefers-reduced-motion: reduce)` block (`admin-css-build.mjs:555`). Durations are capped at
300ms; nothing in admin animates above `motion-transition`.

## Copywriting Contract

This phase ships **no user-facing UI** — it emits tokens, a CI job, and a generalized test
harness. The "copy" surface here is **operator/CI-facing diagnostic text** (the gate's failure
output IS the worklist). Locked in the operator/SRE voice (GDS: say what happened + how to fix;
no please/oops/sorry/jargon):

| Element | Copy |
|---------|------|
| Primary CTA (n/a — CI job) | Job name: `brandbook-tokens` (regen → contrast → gallery-check → diff) |
| Empty state (n/a) | No screen surface this phase; downstream phases own empty states |
| Drift failure (the load-bearing message) | `Generated CSS is out of sync with tokens.json. Run the brandbook generators and commit the result.` (printed before/around `git diff --exit-code`) |
| Contrast failure | `Contrast gate failed: {fg} on {bg} ({theme}) = {ratio}, needs {min}. Adjust the token or pairing.` |
| Missing-token (parity) failure | `Token category {name} emitted but not registered in the parity array. Add it to exact()/requiredTokenUses.` |
| Destructive actions | None this phase. No data mutation, no irreversible operation; the only "destructive" act is failing a build, which is the intended safety behavior. |

## Machine-Verifiable Acceptance (for gsd-ui-checker & gsd-ui-auditor)

The four token categories above are the contract. Verification is mechanical:

1. **Idempotency / drift:** `tokens-build.mjs` → `admin-css-build.mjs` → sync both committed CSS
   copies (`brandbook/tokens/rindle-admin.css` **and** `priv/static/rindle_admin/rindle-admin.css`)
   → `git diff --exit-code` is clean on a single regen (D-94-03, D-94-10).
2. **Contrast:** all new dark status-surface pairs and the elevation `dark-text` pairs are added to
   `CONSOLE_CONTRAST_PAIRS` / `contrast_pairs` and pass at their stated min (values tabulated above).
3. **Parity:** every new category (`elevation`, extended `motion`, fluid `typography`/`spacing`
   steps, `color.semantic.{light,dark}` differentiated status surfaces, `breakpoint`) is registered
   in `exact()` / `requiredTokenUses` and `MOTION_TOKENS`.
4. **Polish harness:** `admin-polish.js` accepts `{ root, interactiveSelectors }` (defaulting to
   `[data-rindle-admin-root]` / `.rindle-admin-*`) so the existing admin spec passes unchanged and
   Cohort can later pass `[data-ck-root]` / `.ck-*` (D-94-07).
