<objective>
Fix the two S3-adapter-local gaps that the reaper and the multi-node resume path
depend on:

- **CR-02 (path source-of-truth):** expose a public `Rindle.Storage.S3.tus_tail_path/2`
  that delegates to the existing private `tail_path/2` so the *adapter itself*
  owns the one canonical tail-path computation (the `Base.url_encode64(id, padding: false) <> ".tail"`
  encoding). Plan 43-08 will route the reaper's `remove_tus_tail/1` through this
  helper, ending the raw-UUID vs base64url mismatch.
- **CR-04 (silent cross-node corruption):** add a loud-fail guard in
  `upload_part_stream/5` — when `state` carries a `multipart_upload_id` and a
  non-empty `parts` list (a mid-multipart resume) but the expected tail file is
  absent on this node, return `{:error, :tus_tail_missing}` instead of silently
  re-slicing from a fresh empty tail (which corrupts the assembled object). Also
  document the single-node / sticky-session deployment constraint in the S3
  moduledoc.

Purpose: This plan owns ALL `lib/rindle/storage/s3.ex` changes for the gap-closure
set, keeping file ownership clean so it can run in parallel with the sweeper plan
(43-07). It is the source-of-truth dependency for the reaper-wiring plan (43-08).

Output: A public tail-path helper, a tagged cross-node guard, an updated moduledoc,
and two RED-then-GREEN regression tests.
</objective>

| PATCH body → S3 adapter | untrusted bytes drained to a node-local tail file, then sliced into S3 parts |
| reaper → filesystem | the reaper deletes tail files under Rindle.tmp/tus/ by computed path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-06-01 | Tampering | upload_part_stream/5 cross-node resume | mitigate | Guard returns `{:error, :tus_tail_missing}` when DB shows mid-multipart but local tail is absent — prevents silent object corruption (the integrity failure CR-04 names). |
| T-43-06-02 | Information disclosure | :tus_tail_missing error term | mitigate | Error surface is the bare atom only; the absolute tail path and session_uri are NOT included in the returned term (no internal-path leak across the adapter boundary). |
| T-43-06-03 | Tampering | tus_tail_path/2 traversal | accept | id is the server-issued session_id (UUID) base64url-encoded under a fixed Rindle.tmp/tus/ prefix — structurally traversal-proof (same guard as Local.tus_part_path/2); helper does not widen the surface. |
</threat_model>
