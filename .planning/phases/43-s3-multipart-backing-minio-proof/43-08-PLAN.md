<objective>
Own ALL `lib/rindle/ops/upload_maintenance.ex` gap fixes and prepare the shared
abort helper the DELETE path (43-09) consumes:

- **CR-02 (wiring):** route `remove_tus_tail/1` through `Rindle.Storage.S3.tus_tail_path/2`
  (added in 43-06) so the reaper deletes the real base64url-encoded tail file
  instead of a raw-UUID path that never matches.
- **IN-03:** `abort_tus_backing/1`'s Local branch calls
  `Rindle.Storage.Local.tus_part_path(session.id, [])` — empty opts resolve the
  part path against the default app-env root, not the upload's actual root. Resolve
  the Local root from the session's profile/adapter config (the same resolution
  `resolve_tus_adapter/1` already performs) before removing the part file.
- **WR-01:** `do_expire_tus_session/2` calls `MediaUploadSession.changeset(.., %{state: "expired"})`
  directly, skipping the `UploadSessionFSM.transition/3` invariant gate that
  `expire_standard_session/2` deliberately enforces. Route tus expiry through the
  FSM gate (or a shared gated-expire helper) so a future query-set expansion can't
  silently violate the FSM contract.
- **CR-01 prerequisite (shared helper):** make the backing-store abort reusable as
  `abort_tus_backing/2` that accepts an explicit adapter+root (and still works in
  the reaper's resolve-from-session mode), so the tus DELETE handler (43-09) can
  invoke the SAME polymorphic abort (S3 multipart abort / Local tmp removal)
  without duplicating logic.

Purpose: This plan depends on 43-06 (consumes `S3.tus_tail_path/2`) and is the
prerequisite for 43-09 (which consumes the shared `abort_tus_backing/2`). It owns
the reaper module so 43-09 only touches the Plug.

Output: An encoding-correct, root-correct, FSM-gated, reusable reaper abort path
with three regression tests.
</objective>
