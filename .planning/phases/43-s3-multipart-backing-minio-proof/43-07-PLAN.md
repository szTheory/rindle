<objective>
Close **CR-03**: `Rindle.tmp/` sweeper only deletes directories, so `tus/*.tail`
and `tus/*.part` regular files accumulate without bound (the `tus/` directory mtime
is refreshed on every write, so the directory itself is never aged out on any active
system).

Fix: make the sweeper recurse into the `tus/` subdirectory and age out individual
regular files there (mtime older than the threshold), as a safety net behind the
explicit end-of-life cleanup (43-08 reaper + 43-09 DELETE). This is the backstop
the S3 `complete_part_stream` best-effort `File.rm`, the DELETE handler, and the
reaper all (incorrectly) rely on today.

Purpose: Independent file ownership (`sweep_orphaned_temp_files.ex`) → runs in
parallel with the S3-adapter plan (43-06) in Wave 1.

Output: A recursing sweeper that ages out `tus/*` regular files, plus a new test
file proving it.
</objective>

  def sweep(opts \\ []) do
    root = Keyword.get(opts, :dir, TempRunDir.root_dir())
    threshold_sec = Keyword.get(opts, :threshold_sec, @default_threshold_sec)
    dry_run? = Keyword.get(opts, :dry_run, true)
    threshold_time = System.system_time(:second) - threshold_sec
    ...
    root |> File.ls!() |> Enum.reduce(base, fn entry, acc ->
      process_run_dir(Path.join(root, entry), threshold_time, dry_run?, acc)
    end)
  end

  defp process_run_dir(path, threshold_time, dry_run?, acc) do
    case File.lstat(path) do
      {:ok, %File.Stat{type: :directory, mtime: mtime}} -> ...delete_run_dir...
      {:ok, _stat} -> acc          # <-- BUG: regular files silently skipped
      {:error, _reason} -> %{acc | errors: acc.errors + 1}
    end
  end
