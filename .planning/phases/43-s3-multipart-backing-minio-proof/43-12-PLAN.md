<objective>
Close **CR-04** completely. Prior work added a cross-node resume guard
(`guard_local_tail_present/2`, s3.ex:289-298) but it only fires when `parts != []`.
A first PATCH under 5 MiB sets `upload_id`, buffers N MiB to the node-local tail,
but commits NO part (`parts: []`, `offset > 0`). If the next PATCH routes to a
different node, the guard sees `parts == []` → returns `:ok`, the fresh node opens
an empty tail and appends from byte 0, silently dropping the first node's buffered
bytes — the assembled object is corrupted with no error returned to the client.

Strengthen the guard so it fires whenever the persisted DB state implies bytes were
buffered on another node — not only after a part commits. The robust signal:
`committed_part_bytes = length(parts) * @s3_min_part_size`; require the tail to exist
whenever `offset > committed_part_bytes`. This catches the pre-first-part window
(`parts == []`, `committed_part_bytes == 0`, `offset > 0`) while preserving the
brand-new FIRST PATCH happy path (`offset == 0`, so `0 > 0` is false) AND the
already-covered committed-part case.

This plan owns ONLY `lib/rindle/storage/s3.ex` + its unit test. It has NO file
overlap with 43-11 (CR-01 / tus_plug.ex + upload_maintenance.ex), so the two run in
the same wave.

Purpose: convert the silent data-corruption window into a loud, safe failure so a
sub-5-MiB-first-chunk cross-node resume never produces a corrupted object — the
integrity half of TUS-09's zero-leak/correctness goal.

Output: a strengthened offset-aware guard, a unit test proving the pre-first-part
hole now fails loudly, and a same-node happy-path test proving no regression.
</objective>

  # CURRENT guard (the hole): mid_multipart? requires parts != [], so parts==[] (pre-first-part) bypasses it.
  defp guard_local_tail_present(tail_path, state) do
    upload_id = Map.get(state, :upload_id)
    parts = Map.get(state, :parts, [])
    mid_multipart? = is_binary(upload_id) and upload_id != "" and is_list(parts) and parts != []
    if mid_multipart? and not File.exists?(tail_path), do: {:error, :tus_tail_missing}, else: :ok
  end

| adapter → caller | the error surface returned across the adapter boundary must not leak internal paths |
