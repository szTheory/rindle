<objective>
Close the headline blocker **CR-01** and the co-located **WR-02**, plus add the
CR-04 Plug-side moduledoc note:

- **CR-01 (the cost leak):** `handle_delete/2` sets `state: "aborted"` and returns
  204 but performs NO backing-store abort, and NO reaper query ever re-selects an
  aborted tus session — so a DELETEd S3-backed tus session permanently orphans its
  S3 multipart upload. Fix: invoke the shared polymorphic backing abort
  (`Rindle.Ops.UploadMaintenance.abort_tus_backing/2`, extracted in 43-08) inside
  `handle_delete/2` BEFORE the state transition, using the adapter + root the Plug
  already holds in `opts` (no DB profile re-resolution). For S3 this aborts the
  multipart; for Local it removes the tmp part/tail. Add the regression test that
  asserts `abort_multipart_upload` is invoked for a DELETEd S3-backed tus session.
- **WR-02:** `handle_delete/2` discards the `Config.repo().update()` result and
  always returns 204. Match on the result and return a 5xx (or tagged error) on
  `{:error, _}` so the client is not falsely told the upload was terminated while
  the row remains in its prior state (re-PATCHable / mis-reaped). CRITICAL: because
  the abort runs BEFORE the update, the WR-02 failure-path test must ALSO assert the
  backing abort already fired — proving the abort precedes the update at runtime, not
  just in source order (CR-01 ordering requirement).
- **CR-04 (Plug moduledoc half):** add the single-node / sticky-session constraint
  note to the TusPlug moduledoc (the S3 adapter half lands in 43-06), since the
  Plug is the adopter-facing mount point.

Purpose: Depends on 43-08 (consumes the shared `abort_tus_backing(session, opts)`).
Owns ONLY `lib/rindle/upload/tus_plug.ex` so it serializes cleanly after the reaper
module plan. This is the plan that makes SC4 (DELETE no-leak) achievable; SC5's
DELETE MinIO proof (43-10) depends on it.

Output: A DELETE handler that aborts the backing store BEFORE the state transition
and honours the update result, the Plug single-node moduledoc, and three regression
tests.
