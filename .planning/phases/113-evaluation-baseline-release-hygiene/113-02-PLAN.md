| GitHub Actions secret -> release pipeline | RELEASE_PLEASE_TOKEN crosses from repo secret store into the release-please job; a present-but-invalid value silently defeats the `|| github.token` fallback. |
| scheduled workflow -> GitHub issues API | the drift guard writes/closes issues using least-privilege issues:write. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-113-03 | Spoofing / DoS-of-release | `secrets.RELEASE_PLEASE_TOKEN \|\| github.token` masking a bad token | high | mitigate | D-06b token-validity step runs `gh api user` and fails loud with a rotation hint before Run Release Please, so an invalid token cannot silently stall the release train. |
| T-113-04 | Information Disclosure | secret leakage in workflow logs | high | mitigate | Guard reads the token via `env:` only, checks `gh api user` exit code, never prints/echoes the token value; drift guard uses github.token, not the release PAT. |
| T-113-05 | Elevation of Privilege | over-scoped guard-workflow token | medium | mitigate | release-train-drift permissions pinned to `contents: read` + `issues: write` only; no write to contents/PRs. |
| T-113-06 | Tampering (release-coupling) | a new required check sneaking onto CI Summary | high | mitigate | D-09: guards stay OFF ci-summary needs:; the new release_guard_meta_test.exs asserts this and runs in mix ci, mirroring how ci_observability_test.exs locks OBS-02. |
| T-113-SC | Tampering | npm/pip/cargo installs | n/a | accept | No package installs — the workflow reuses already-pinned actions (checkout, JasonEtco) and gh/curl already in the pipeline. |
