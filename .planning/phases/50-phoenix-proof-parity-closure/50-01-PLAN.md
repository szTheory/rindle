<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Generated LiveView helper -> browser uploader | The generated app must emit the same `RindleTus` metadata contract documented for adopters. |
| Browser transfer -> server readiness | Upload byte progress must not be mistaken for verified asset readiness. |
| Proof report JSON -> CI/pass/fail judgment | Missing or prose-only Phoenix-facing fields can create false confidence that the documented path is proven. |

| T-50-01-02 | R | `install_smoke_tus_report.json` | mitigate | Persist explicit Phoenix-facing keys and the exact `completion_surface` marker so proof claims remain auditable. |
| T-50-01-03 | T | generated-app smoke assertions | mitigate | Keep the existing transport/runtime assertions while adding explicit Phoenix-path and honest-state assertions, so the proof cannot silently downgrade back to raw transport-only evidence. |
</threat_model>
