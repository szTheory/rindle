# Plan 02-01: Phoenix Controller Proxied Upload

**Goal:** Implement server-side proxied upload support for standard Phoenix controllers, enabling multipart uploads that stream directly to storage.

## Success Criteria
- [ ] `Rindle.upload/2` implemented as the primary entry point for proxied uploads.
- [ ] Support for `Plug.Upload` structs (standard Phoenix controller params).
- [ ] File data is streamed from the temporary path to the storage adapter.
- [ ] Uploaded files undergo magic-byte MIME detection and validation.
- [ ] Unit/Integration tests for `Rindle.upload/2`.

## Proposed Changes

### Public API
- Update `lib/rindle.ex` to include `upload/2`.

### Core Logic
- Implement `Rindle.upload/2` to:
    1. Detect MIME from magic bytes.
    2. Validate against profile policy.
    3. Generate storage key.
    4. Call `storage.store/3` (streaming is handled by adapter/File.cp).
    5. Create `MediaAsset` in `validating` or `analyzing` state.

### Testing
- Create `test/rindle/upload/proxied_test.exs`.
- Test with valid and invalid (quarantined) files.

## Verification Plan

### Automated Tests
- Run `mix test test/rindle/upload/proxied_test.exs`.
