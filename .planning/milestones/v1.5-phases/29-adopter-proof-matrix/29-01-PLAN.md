<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generated app -> installed package | The proof must use the packaged/public artifact and not silently fall back to repo-local dependencies. |
| generated app -> migration/runtime wiring | Incorrect repo or migration resolution would create a false-positive install proof. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-01 | Spoofing | generated-app dependency source | mitigate | Assert `deps/rindle` is absent and network mode or package-root mode is explicit in the generated-app report. |
| T-29-02 | Repudiation | install smoke evidence | mitigate | Keep proof in ExUnit with explicit report fields for compile, boot, migration, and lifecycle success. |
| T-29-03 | Tampering | migration handoff | mitigate | Preserve explicit `Application.app_dir(:rindle, "priv/repo/migrations")` resolution in the generated-app helper and smoke assertions. |
</threat_model>
