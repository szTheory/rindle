# 82-01: Generation pipeline and five candidate SVGs

## Objective

Build the reproducible SVG generation pipeline (opentype.js text→path outlining of the
Space Grotesk Medium wordmark; no viewer font dependency) and author five genuinely
distinct logo direction candidates honoring the locked constraints: no containers, tight
logotype, no subtitle on main lockups, ≥2 fully integrated typemarks.

## Tasks

1. `brandbook/src/` pipeline: `geometry.mjs` (constants), `candidates.mjs` (five
   concepts, tunable geometry), `build.mjs` (outline + compose + write variants),
   `render.mjs` (Playwright rasterizer, reuses examples/adoption_demo install),
   `check.mjs` (constraint + budget gates). OFL TTFs committed in `src/fonts/`.
2. Five candidates × (light, dark, mono, mark, mark-dark, subtitle) =
   30 SVGs in `brandbook/assets/logo/candidates/`.
3. Visual iteration loop (build → Playwright screenshot → review) until each candidate
   reads its concept: A Headgate, B Headwater r, C Branch dot, D Outflow e, E Confluence.

## Verification

- `node check.mjs` green (viewBox, no text/image/rect, no external refs, size budgets).
- `xmllint --noout` on all candidate SVGs.
- B, C, D are integrated typemarks (motif worked into letterforms) — exceeds the ≥2 bar.

## Deviations

- Candidate A's concept changed during iteration from "Meander" (S-bend channel — read
  as a snake/duck at small sizes across three geometry attempts) to "Headgate" (open
  arch sheltering a current that flows through it). Same Route-A channel lineage, more
  literal rendering of the locked brand idea "a protected current".
- `opentype.js` `toPathData()` output silently truncates in Chromium mid-wordmark;
  replaced with our own command serializer in `build.mjs`.
