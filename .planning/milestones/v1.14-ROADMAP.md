# Milestone v1.14: Bulk Owner-Erasure Orchestration

**Status:** ✅ SHIPPED 2026-05-27
**Phases:** 67–70
**Total Plans:** 8

## Overview

Extend the shipped v1.10 single-owner erasure facade with batch preview/execute,
an operator mix surface, and hermetic proof + docs parity — policy-first, reusing
`OwnerErasure` internals without bundling force-delete or admin UI.

## Phases

### Phase 67: Bulk erasure policy & contract

**Goal:** Freeze the batch erasure boundary before implementation lands.
**Depends on:** Phase 66 (v1.13)
**Plans:** 2 plans

Plans:

- [x] 67-01: Batch owner-erasure public contract (types, boundary stubs, contract tests)
- [x] 67-02: Batch erasure error vocabulary and api_surface_boundary freeze

### Phase 68: Batch erasure implementation

**Goal:** Implement batch preview/execute reusing `OwnerErasure` with per-owner isolation.
**Depends on:** Phase 67
**Plans:** 2 plans

Plans:

- [x] 68-01: Sequential batch preview/execute orchestration on `Rindle` facade

**Requirements:** BULK-03, BULK-04, BULK-05

## Milestone Summary

**Key Decisions:**

- v1.14 bulk erasure extends v1.10 facade; batch orchestration reuses `OwnerErasure`.
- Contract-before-implementation: Phase 67 froze types/errors before planner wiring.
- No force-delete, admin UI, or scheduler workflows in v1.14 scope.
- PROOF-05 hermetic matrix is API-only by design; operator CLI covered separately (D-12).

**Issues Resolved:**

- Admin/bulk owner-erasure orchestration gap from v1.10/v1.13 deferrals.
- Multi-owner compliance workflows no longer require hand-rolled per-owner loops.
- Phase 68 partial-failure DB gap closed by Phase 70 `CountingFailingTxnRepo` proofs.

**Issues Deferred:**

- Force-delete policy for assets with surviving attachments (v1.15+ demand).
- Mix `batch_owner_failed` partial-report E2E test (code present; flow gap non-blocking).
- Nyquist validation hygiene on phases 68–70 (`/gsd-validate-phase` discovery only).

**Technical Debt Incurred:**

- Mix task `batch_owner_failed` path lacks integration test driving partial report + exit 1.
- CLI `--max-owners` flag forwarding not exercised end-to-end in task tests.
- Batch preview partial-failure halt shares execute path; only execute has full DB proof.
