## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions token → Actions API | The aggregator reads run/job metadata with the run token |
| fork PR → CI workflow | Skip-prone secret/label-gated jobs must not turn a fork skip into a failure |
| CI run → uploaded artifacts | JUnit/coverage artifacts are retrievable for the retention window |
| workflow permissions → individual jobs | A workflow-level scope widening would over-privilege every job |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-103-06 | Elevation of Privilege | `actions: read` permission creep | mitigate | Declared at the `ci-observability` job level ONLY (precedent release.yml:101-103); acceptance criteria asserts workflow-level `permissions` stays `{contents: read}` (ASVS V4) |
| T-103-07 | Denial of Service / Information Disclosure | Aggregator turning fork-PR skips into failures or leaking secret-gated job status | mitigate | `mux-soak`/`gcs-soak`/`package-consumer-gcs-live` OMITTED from `needs:`; `if: always()`; advisory-only, never a required check (Pitfall 4, ASVS V14) |
| T-103-08 | Information Disclosure | JUnit/coverage artifacts leaking secrets | mitigate | Artifacts contain test names + coverage %, no secrets; bounded `retention-days: 14` |
| T-103-09 | Tampering | Adding the aggregator to required checks / changing a `run:` gate | mitigate | Aggregator never added to branch protection (D-14, Phase 105 scope); `mix coveralls` stays the gating step; Plan 04 diff proves required-check set unchanged |
| T-103-SC | Tampering | npm/pip/cargo installs in this plan | accept | No package installs in this plan — only YAML workflow edits; supply-chain surface is the existing pinned actions (SHA-pinning is Phase 107 HARD-02, out of scope) |
