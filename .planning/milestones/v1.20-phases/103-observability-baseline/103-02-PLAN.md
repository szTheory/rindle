<objective>
Create the two OBS-03 maintainer-local collector scripts under `scripts/ci/`: one that computes a
reproducible historical timing baseline (per-job avg + p95 + rerun rate) over recent `ci.yml` runs,
and one that records the *live* branch-protection required-check names and diffs them against the
committed expected list. Both are read-only `gh api` tooling. This is Wave 0 / harness work — the
scripts must exist and lint-pass before Plan 04 runs them to author `103-BASELINE.md`.

"regression vs baseline" check has a real reference, and a verified live-vs-expected required-check
capture that the Phase 105 branch-protection flip depends on. Live drift already exists today
(`brandbook-tokens` is in the expected list but absent from live `.contexts[]`) — capturing it
verbatim is the whole point.

Output: `scripts/ci/collect_ci_baseline.sh` and `scripts/ci/check_required_checks.sh`, both passing
`bash -n`, both matching the existing `scripts/ci/` house style.
</objective>
