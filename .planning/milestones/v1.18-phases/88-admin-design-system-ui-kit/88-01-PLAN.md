<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| token JSON -> generator | Token names and values drive generated CSS and contrast checks. |
| generator -> generated CSS | CSS will be consumed by future admin UI and later packaged as a library-owned asset. |
| generated CSS -> host app | Host app pages may include the CSS; selectors must stay namespaced and self-contained. |
| maintainer scripts -> local filesystem | Node scripts write generated assets under `brandbook/`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-88-01 | Tampering | `brandbook/tokens/rindle-admin.css` | mitigate | Generated header plus `admin-css-build.mjs` parity checks; task verifies regeneration from `tokens.json` and forbids hand-edited-only output. |
| T-88-02 | Tampering | CSS dependency boundary | mitigate | Static scan blocks Tailwind, daisyUI, shadcn, Radix, `@apply`, Tailwind UI, daisyUI registry, and host asset-pipeline references. |
| T-88-03 | Information Disclosure | lifecycle/status components | mitigate | Status styles require visible labels/non-color marks and contrast pairs for ready, processing, warning, danger, quarantine, and info. |
| T-88-04 | Denial of Service | future non-console adopters | mitigate | Phase keeps work in Node/static assets and does not add LiveView aliases, router macros, plugs, or package dependency changes. |
| T-88-05 | Spoofing | theme/status semantics | mitigate | `THEMES` and `STATUS_STATES` are allowlisted; generator fails missing or unknown state selectors. |
| T-88-SC | Tampering | npm installs | accept | No package-manager install task is planned; existing Playwright/opentype packages were audited in research and no new package is added. |
</threat_model>
