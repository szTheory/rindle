## Design System

| Property | Value |
|----------|-------|
| Tool | none; Phase 87 changes Docker preview, shell output, and docs only |
| Preset | not applicable |
| Component library | none; do not add rendered UI components in this phase |
| Icon library | none; terminal output and docs use text labels, not icons |

Design-system contract:

- Phase 87 must not introduce shadcn, Radix, Tailwind UI, daisyUI registry components, or any third-party UI registry.
- Phase 87 must not implement the admin console, Cohort rebrand, theme picker, UI kit, or component gallery. Those remain scoped to phases 88-91.
- The only user-facing interaction surface added in Phase 87 is launch output from `scripts/demo/up.sh` plus matching Docker quick-try documentation.
- The printed admin console URL is a stable affordance for later phases, even before the route renders a console.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps in docs tables and command examples |
| sm | 8px | Compact spacing between command labels and code snippets |
| md | 16px | Default docs paragraph, list, and table rhythm |
| lg | 24px | Section spacing in Docker quick-try docs |
| xl | 32px | Major docs subsection breaks |
| 2xl | 48px | Page-level docs grouping only |
| 3xl | 64px | Not used by Phase 87 output; reserved for later rendered UI |

Exceptions:

- `12px` may be used only where existing brand tokens or markdown rendering already use it for dense table/cell rhythm.
- Interactive target minimum `44px` is not applicable to Phase 87 because no new rendered controls are introduced.
- Terminal output must use one blank line before the URL map and one blank line after it so the copy-paste targets are visually isolated.

Source: `brandbook/tokens/tokens.json`; `87-CONTEXT.md` D-87-06 and D-87-07; default 8-point contract.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Start demo |
| Empty state heading | No demo stack is running |
| Empty state body | Run `./scripts/demo/up.sh` to start Cohort, then open the app, admin console, or MinIO console URL from the launch output. |
| Error state | Docker preview could not start. Check whether ports `4102`, `9000`, or `9001` are busy, then rerun with `COHORT_DEMO_PORT`, `COHORT_MINIO_PORT`, or `COHORT_MINIO_CONSOLE_PORT`. |
| Destructive confirmation | none in Phase 87; `scripts/demo/reset.sh` keeps existing volume-reset behavior and must not gain a new destructive UI flow |

Copywriting contract:

- Launch output must print exactly these labels: `app`, `admin console`, and `MinIO console`.
- Launch URLs must resolve from the same env-driven defaults Compose uses:
  `http://localhost:${COHORT_DEMO_PORT:-4102}`,
  `http://localhost:${COHORT_DEMO_PORT:-4102}/admin/rindle`, and
  `http://localhost:${COHORT_MINIO_CONSOLE_PORT:-9001}`.
- Port-conflict guidance must be action-first: name the busy default port, name the env var to override, and avoid telling maintainers to edit the compose file.
- Do not present the Docker preview as production deployment, public MinIO exposure, or a security boundary.
- Keep docs wording reader-empathetic and copy-pasteable: commands first, explanation second.

Source: `87-CONTEXT.md` D-87-01 through D-87-14; `87-RESEARCH.md` Phase Requirements; `guides/docker_demo_dx.md`.

## Interaction Contract

Phase 87's interaction surface is terminal-first Docker preview startup plus matching docs. The contract is about clarity, copyability, and contract drift prevention between Compose, shell output, runtime env, and docs.

| Area | Contract |
|------|----------|
| Launch entry point | Preserve `./scripts/demo/up.sh` as the primary copy-paste command. It may accept existing compose passthrough args, but must not require raw `docker compose` knowledge for the happy path. |
| URL map | Print a scannable URL map with labels `app`, `admin console`, and `MinIO console`. Each URL appears on its own line and is copy-pasteable without surrounding punctuation. |
| Env overrides | `COHORT_DEMO_PORT`, `COHORT_MINIO_PORT`, `COHORT_MINIO_CONSOLE_PORT`, and `COMPOSE_PROJECT_NAME` are the user-facing override controls. Docs must show examples using env vars, not compose-file edits. |
| Namespacing | `COMPOSE_PROJECT_NAME` is the sibling-stack collision escape hatch. Wrapper docs must explain that it scopes containers, networks, and volumes. |
| Port conflicts | Failure guidance names the likely occupied default and the matching override variable. Do not add process-killing or custom port scanning UX. |
| Exposure | Published app and MinIO ports remain loopback-bound. MinIO console is described as local developer tooling only. |
| Admin URL | Always print `http://localhost:${COHORT_DEMO_PORT}/admin/rindle` as `admin console`, even while the later console route is not yet implemented. |
| Cache expectations | Docs must state that routine source, style, and template edits should not refetch Hex dependencies after the Dockerfile ordering fix. |
| Verification feedback | Static verification should prefer rendered `docker compose config`, shell syntax/static checks, and targeted launch-output checks before heavier full-stack startup. |
| Accessibility | Docs and terminal output must not depend on color, hover disclosure, or icons. URLs and env vars must be visible as text. |

Visual hierarchy contract:

- The primary focal point in launch output is the URL map.
- The secondary information is port/project override guidance.
- Error and recovery copy must name the failed operation and next command-level action.
- Do not add decorative banners, ASCII art, spinners, or animation to the shell output.

Source: `87-CONTEXT.md`; `87-RESEARCH.md` Architecture Patterns and Common Pitfalls; `guides/docker_demo_dx.md`.
