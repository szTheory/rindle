# Phase 91: Code Review Report

**Reviewed:** 2026-06-12T21:42:08Z
**Depth:** standard
**Files Reviewed:** 4
**Status:** issues_found

## Summary

The review evaluated the media profiles, lifecycle seed data, admin console routing, and documentation updates for the Cohort demo. While the structural requirements for the profiles are met, the seed script bypasses domain invariants and creates malformed `MediaAsset` records by stuffing PDFs into the `"image"` kind. Additionally, there is a path mismatch between the admin console route and the documentation, and a pre-existing configuration evaluation bug in the router.

## Critical Issues

### CR-01: Invalid Ecto invariants generated by lifecycle seed data

**File:** `examples/adoption_demo/priv/repo/seeds.exs:144-148`
**Issue:** The seed loop creates edge cases for `DocumentProfile` (`application/pdf`), but forces the asset's `kind` to `"image"`. Rindle's `MediaAsset` schema only natively supports `image`, `video`, and `audio` kinds. Bypassing the schema validation with `Repo.insert!` creates structurally corrupt data. The UI and any background media processors expecting an `"image"` kind will crash when attempting to process a PDF.
**Fix:** Since Rindle does not support a `"document"` kind in core invariants, seed edge cases for valid image profiles (like `AdoptionDemo.RindleProfile`) instead of spoofing documents.
```elixir
    kind = if rem(idx, 2) == 0, do: "audio", else: "image"
    
    {profile, content_type} =
      if kind == "audio" do
        {"AdoptionDemo.AudioProfile", "audio/mpeg"}
      else
        {"AdoptionDemo.RindleProfile", "image/jpeg"}
      end
```

## Warnings

### WR-01: Documentation mismatch with Rindle Admin route

**File:** `examples/adoption_demo/lib/adoption_demo_web/router.ex:35` (and `examples/adoption_demo/README.md:21`)
**Issue:** The `README.md` documents the admin console as being available at `http://localhost:4102/admin/rindle`. However, the router mounts `rindle_admin "/"` within `scope "/admin"`, making the actual path `http://localhost:4102/admin`. Users following the README will receive a 404.
**Fix:** Mount the route exactly where the documentation advertises it.
```elixir
  scope "/admin" do
    pipe_through :browser

    rindle_admin "/rindle", allow_unauthenticated?: true
  end
```

### WR-02: Compile-time evaluation of `secret_key_base` in router

**File:** `examples/adoption_demo/lib/adoption_demo_web/router.ex:14-17`
**Issue:** `Application.compile_env!` is evaluated during the compilation of the router module for the `TusPlug`. This bakes the build-time `secret_key_base` into the bytecode, meaning runtime changes via `runtime.exs` or environment variables will be ignored, leading to failed upload signatures or security weaknesses.
**Fix:** Use a lazy configuration resolution approach (like a runtime module lookup) or wrap the plug configuration to fetch at runtime if supported by `TusPlug`.
```elixir
  # Ensure the configuration is deferred until runtime, for example:
  forward "/uploads/tus", Rindle.Upload.TusPlug,
    profile: AdoptionDemo.VideoProfile,
    endpoint: AdoptionDemoWeb.Endpoint
```
