<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser params -> LiveView | Route params and filter params enter LiveView mounts and params handlers. |
| LiveView -> admin queries | LiveViews request read models from `Rindle.Admin.Queries`. |
| PubSub -> LiveView | PubSub events from lifecycle workers invalidate visible data. |
| query data -> browser | Redacted read models render into operator-visible HTML. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-89-16 | Tampering | PubSub payload handling | mitigate | Treat payloads only as invalidation; tests broadcast forged secret payloads and assert query refresh output. |
| T-89-17 | Information Disclosure | upload-session detail | mitigate | Render redacted session URI copy from `Rindle.Admin.Queries`; tests assert raw URI absence. |
| T-89-18 | Spoofing | active navigation/theme state | mitigate | Use exact surface allowlist, `aria-current="page"`, and `data-theme="light|dark|auto"` controls. |
| T-89-19 | Denial of Service | LiveView subscriptions | mitigate | Subscribe only to visible detail/list row topics, not broad wildcard or second channel topics. |
| T-89-20 | Elevation of Privilege | read surfaces | mitigate | No destructive or repair calls; first three surfaces are read-only and link only to investigation routes. |
| T-89-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install task is planned; LiveViews consume existing packaged CSS/JS. |
</threat_model>
