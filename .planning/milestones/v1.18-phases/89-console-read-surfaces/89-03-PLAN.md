<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LiveView/UI -> admin queries | Untrusted route params and filter params will be normalized before DB reads. |
| admin queries -> database | Query module reads domain schemas through adopter-configured repo. |
| domain rows -> operator UI | Sensitive provider/upload internals must be redacted before rendering. |
| admin queries -> public facade | Query helpers must not leak into `lib/rindle.ex`. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-89-12 | Information Disclosure | provider asset summaries | mitigate | Use `MediaProviderAsset.redact_id/1` or `Provider identifier redacted`; tests assert raw provider IDs are absent. |
| T-89-13 | Tampering | query filters | mitigate | Normalize allowlisted filters and reject unknown keys with tagged errors. |
