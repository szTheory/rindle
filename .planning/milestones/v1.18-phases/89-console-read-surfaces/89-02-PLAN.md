<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| brandbook generated CSS -> package static CSS | Generated artifact is copied into runtime package assets. |
| host request -> `Plug.Static` | Host routes serve library-owned static files to browsers. |
| package files -> Hex consumer | Hex consumers receive only paths listed in `mix.exs` package metadata. |
| admin JS -> host DOM | Self-contained JavaScript mutates only admin theme attributes. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-89-07 | Information Disclosure | Static asset route | mitigate | Test `Plug.Static` allowlist denies traversal and unlisted files. |
| T-89-08 | Tampering | Hex package metadata | mitigate | Extend package metadata tests using real `mix hex.build --unpack` output. |
| T-89-09 | Spoofing | Theme JavaScript | mitigate | Restrict `data-theme` writes to exact `light`, `dark`, and `auto` values and scope behavior to `[data-rindle-admin-root]`. |
| T-89-10 | Denial of Service | Host asset pipeline | mitigate | Serve precompiled self-contained assets from `:rindle`; no host Tailwind/esbuild pipeline dependency. |
| T-89-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install task is planned; all assets come from existing repo files. |
