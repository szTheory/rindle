<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| maintenance service -> storage adapter | Remote resumable cancel crosses into an external bearer-credential boundary and must stay outside DB transactions. |
| service report -> worker/task surfaces | Bounded operator details must cross into logs, telemetry, and CLI output without leaking `session_uri` or raw provider payloads. |
| abort worker -> Oban retry engine | Retry behavior depends on the worker surfacing non-idempotent per-session failures as job failure rather than reporting `:ok`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-01 | I | `lib/rindle/ops/upload_maintenance.ex` | mitigate | Map resumable cancel failures to bounded `failure_reason` strings per D-02 and D-03; never log or persist raw `session_uri` or provider bodies. |
| T-40-02 | T | abort-lane ownership | mitigate | Keep remote resumable cancel only in `abort_incomplete_uploads/1` per D-05 and reject any routing through cleanup or broker-facing cancel helpers. |
| T-40-03 | D | abort worker retry boundary | mitigate | Return worker failure when `abort_errors > 0` and add tests proving Oban-visible failure instead of silent `:ok`. |
| T-40-04 | E | transaction boundary | mitigate | Keep adapter cancel calls outside DB transactions per D-08 and assert this path through service-level tests. |
</threat_model>
