  <action>
    Create `lib/mix/tasks/rindle.batch_owner_erasure.ex` with:

    ```elixir
    defmodule Mix.Tasks.Rindle.BatchOwnerErasure do
      @shortdoc "Preview or execute batch owner-erasure for a JSON owners file"
    ```

    `@moduledoc` MUST include these sections (canonical CLI reference per D-12):
    - `## Usage`: `mix rindle.batch_owner_erasure --owners-file PATH [--dry-run | --no-dry-run | --execute] [--format text|json] [--max-owners N]`
    - `## Options`: document `--owners-file` (required), `--dry-run`, `--no-dry-run`, `--execute` (alias for destructive), `--format`, `--max-owners`
    - `## Owners file format`: JSON array example:
      ```json
      [
        {"owner_type": "Elixir.MyApp.User", "owner_id": "11111111-2222-3333-4444-555555555555"}
      ]
      ```
    - `## Exit codes`: `0` success, `1` any error (including partial batch failure after printing partial report)
    - `## Safety default`: preview unless `--no-dry-run` or `--execute`
    - `## Examples`: preview default, explicit `--dry-run`, destructive `--execute`, `--format json`
    - Links to `guides/operations.md` and `guides/user_flows.md` (cross-link only; Phase 70 adds body)

    `use Mix.Task` with `@requirements ["app.start"]`.

    `OptionParser.parse/2` strict keys:
    `owners_file: :string, dry_run: :boolean, no_dry_run: :boolean, execute: :boolean, format: :string, max_owners: :integer`

    Require `--owners-file`:
    ```elixir
    unless owners_file = Keyword.get(opts, :owners_file) do
      Mix.shell().error("--owners-file PATH is required. See `mix help rindle.batch_owner_erasure`.")
      exit({:shutdown, 1})
    end
    ```

    Dry-run resolution (D-06, D-07):
    ```elixir
    dry_run? =
      case Keyword.fetch(opts, :dry_run) do
        {:ok, value} -> value
        :error -> not (Keyword.get(opts, :execute, false) || Keyword.get(opts, :no_dry_run, false))
      end
    ```

    Format: default `"text"`; reject unknown with message and exit 1.

    Batch opts: `[max_owners: n]` only when `--max-owners` provided.

    `parse_owners_file/1`:
    - `File.read/1` → Jason.decode → must be non-empty list
    - Each entry `%{"owner_type" => type, "owner_id" => id}` (string keys only)
    - `Ecto.UUID.cast(id)` must succeed
    - `resolve_owner_module/1` uses `String.to_existing_atom/1` + `Code.ensure_loaded/1`; on failure print operator error and `exit({:shutdown, 1})`
    - Build `struct(mod, id: uuid)` (no DB fetch)

    On empty list after parse: error "Owners file must contain at least one owner entry." and exit 1 BEFORE calling facade.

    Run batch:
    ```elixir
    runner = if dry_run?, do: &Rindle.preview_batch_owner_erasure/2, else: &Rindle.erase_batch_owner_erasure/2
    ```

    Result handling:
    - `{:ok, report}` → print via `print_report/3` (text or json) → exit 0
    - `{:error, {:batch_owner_failed, detail}}` → print `detail.partial_report` first → `Mix.shell().error(Rindle.Error.message(%{reason: {:batch_owner_failed, detail}}))` → exit 1
    - `{:error, reason}` → `Mix.shell().error(Rindle.Error.message(%{reason: reason}))` → exit 1

    Text output (`print_text_report/2`):
    - Prefix `[DRY RUN] ` when preview mode
    - Lines: owner count (`length(report.owners)`), `attachments_to_detach.count`, `assets_to_purge.count`, `retained_shared_assets.count`
    - Optional brief per-owner line: `owner_type:owner_id` from each entry in `report.owners`

    JSON output: `Mix.shell().info(Jason.encode!(report, pretty: true))`

    Export `@doc false` helpers `format_text_report/2` and `parse_owners_entries/1` (or equivalent pure functions) for testability — mirror `RuntimeStatus.format_text_report/1` pattern.
