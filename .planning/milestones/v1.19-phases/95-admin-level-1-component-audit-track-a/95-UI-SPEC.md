## Level-1 Component Inventory

Phase 95 audits and hardens only these generated admin primitives:

| Component | Required selectors / attributes | Notes |
|-----------|--------------------------------|-------|
| Shell | `.rindle-admin-shell`, `.rindle-admin-shell__main`, `[data-rindle-admin-root]` | Theme scope and page container. |
| Nav | `.rindle-admin-nav`, `.rindle-admin-nav__item`, `[data-rindle-admin-nav-item]` | `aria-current="page"` is active state. |
| Table | `.rindle-admin-table`, `.rindle-admin-table__head`, `.rindle-admin-table__row`, `.rindle-admin-table__cell` | Row hover, row focus-within, empty rows, skeleton rows. |
| Status chip | `.rindle-admin-status-chip`, `.rindle-admin-status-chip--ready|processing|warning|danger|quarantine|info` | Always label plus non-color mark. |
| Button | `.rindle-admin-button`, `--primary`, `--secondary`, `--quiet`, `--destructive`, `[data-rindle-admin-submit]` | Default, hover, focus-visible, active, disabled, loading. |
| Theme picker | `.rindle-admin-theme-picker`, `.rindle-admin-theme-picker__option`, `[data-rindle-admin-theme]` | `aria-pressed="true"` is active state. |
| Form controls | `[data-rindle-admin-input]`, `[data-rindle-admin-confirm-input]` | Inputs, textareas, selects, checkboxes. |
| Confirm dialog | `.rindle-admin-confirm-dialog` | Destructive preview and typed confirmation states. |
| Drawer | `.rindle-admin-drawer` | Overlay surface using dark elevation token in dark theme. |
| Toast | `.rindle-admin-toast`, `--success`, `--warning`, `--danger`, `--info` | Real state change only; no decorative toast motion. |
| Empty/error state | `.rindle-admin-empty-state`, `[data-rindle-admin-empty-state]`, `[data-rindle-admin-error-state]` | Empty and error must share primitive styling but distinct copy. |
| Skeleton | `.rindle-admin-skeleton`, `[data-rindle-admin-loading-state]` | Reserves layout while real pending work resolves. |

State matrix: every interactive or stateful primitive above must be represented in the gallery across `default`, `hover`, `focus-visible`, `active`, `disabled`, `loading`, `empty`, `error`, and `skeleton` as applicable, under `data-theme="light"`, `data-theme="dark"`, and `data-theme="auto"`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs / `--rindle-space-1` | 4px | Tiny inline gaps and picker padding |
| sm / `--rindle-space-2` | 8px | Compact control gaps |
| md / `--rindle-space-4` | 16px | Default element spacing |
| lg / `--rindle-space-5` | 24px | Panel padding and stacked form rhythm |
| xl / `--rindle-space-6` | 32px | Main layout gaps, empty-state padding |
| 2xl / `--rindle-space-7` | 48px | Major section breaks |
| 3xl / `--rindle-space-8` | 64px | Page-level spacing |

Exceptions: `--rindle-space-3` is 12px for table cell padding; `--rindle-admin-target-min` is 44px minimum for interactive controls; `--rindle-space-fluid-gutter` and `--rindle-space-fluid-section` remain the only fluid spacing tokens.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 17px | 400 | 1.6 |
| Label / small | 14px | 600 for controls and chips, 400 for secondary text | 1.45 |
| Heading | 22px | 600 | 1.25 |
| Display | 30px | 600 | 1.15 |

Use exactly these four phase roles for Level-1 components. Larger `h1`/hero token sizes stay page-composition territory for Phase 98, not a new Level-1 requirement.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `--rindle-surface` `#F7F4EA`; dark `--rindle-surface` `#0E1316` | Shell background and main page field |
| Secondary (30%) | Light `--rindle-surface-raised` `#FBFEFC`; dark `--rindle-elevation-1..3` `#161E23`, `#1E282E`, `#27343B` | Nav, cards, tables, drawers, dialogs, toasts |
| Accent (10%) | Light brand `#123A35`; dark brand/accent `#32D08C` with dark-safe foreground | Primary CTA, active theme option, active nav/action state border, focus ring |
| Destructive | Light `#C83232`; dark `#F09090` | Destructive buttons, danger toasts/chips, destructive confirmation only |

Accent reserved for: primary button fill, `aria-pressed="true"` theme option, current/active nav or action affordance, token-backed focus ring, and status-ready/accent marker where paired with text. Do not use `rindle-green` as text on light surfaces; it is accent/large-graphic-only there.

## Interaction Contract

| State | Contract |
|-------|----------|
| Default | Token-backed background, text, border, radius, font, and min target. No page-local color literals. |
| Hover | Surface change or brand-hover token only; no layout shift and no hover-only information. |
| Focus-visible | Every interactive selector uses `outline: var(--rindle-focus-width) solid var(--rindle-focus-ring)` plus `outline-offset: var(--rindle-focus-offset)`. No bare `outline:none`. |
| Active | Pressed/current state is visually distinct from focus: use `transform: translateY(1px)`, `aria-current`, `aria-pressed`, `.active`, or token-backed active fill/border. Never reuse focus ring as active state. |
| Disabled | `disabled` and `aria-disabled="true"` use disabled surface/text tokens, `cursor: not-allowed`, and no press transform. |
| Loading | Real pending work only. Buttons may say `Loading`; skeletons reserve 44px rows; reduced motion sets animation/transition duration to `0ms`. |
| Empty | Empty state says what is absent and the next useful diagnostic action. |
| Error | Error state names the failed surface/artifact and gives a retry or Runtime/Doctor path. |
| Skeleton | Skeletons must have stable dimensions, non-text contrast-safe boundaries, and `aria-hidden="true"`. |

Required interactive selector set for the polish gate: `[data-rindle-admin-submit]`, `[data-rindle-admin-input]`, `[data-rindle-admin-theme]`, `[data-rindle-admin-nav-item]`, `.rindle-admin-actions-tab`, `[data-rindle-admin-detail-link]`, `[data-rindle-admin-action]`, and `.rindle-admin-button`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Review component gallery` |
| Empty state heading | `No assets match this state` |
| Empty state body | `Adjust the lifecycle filter or check Runtime/Doctor for setup issues before running actions.` |
| Error state | `Rindle Admin could not load this surface. Retry load or check Runtime/Doctor for the missing source artifact.` |
| Destructive confirmation | Owner erasure: `Type the owner identifier to confirm after reviewing affected assets.` Batch erasure: `Type the required confirmation after reviewing every affected owner and retained shared asset.` |

Voice: terse operator/SRE copy. Say what happened and what to do next. Do not use "please", "oops", "sorry", marketing language, or color-only labels.
