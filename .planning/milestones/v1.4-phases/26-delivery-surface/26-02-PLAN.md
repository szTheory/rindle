<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| signed playback URL -> plug request | Untrusted clients can tamper with token, range, and key inputs before the plug touches the filesystem. |
| resolved key -> local filesystem path | Key-based resolution can escape the intended root unless path containment is explicit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-04 | Spoofing | `lib/rindle/delivery/local_plug.ex` | mitigate | Verify signed tokens over `key + expiry + actor_subject` with `Plug.Crypto`; reject invalid or expired payloads before file lookup. |
| T-26-05 | Information Disclosure | `lib/rindle/storage/local.ex` | mitigate | Expose one canonical root/path resolver and require `LocalPlug` to reject any resolved path escaping that root per D-12. |
| T-26-06 | Denial of Service | `lib/rindle/delivery/local_plug.ex` | mitigate | Support single-range only and deliberately fall back to `200 + full body` for multi-range or malformed headers rather than attempting multipart assembly. |
</threat_model>
